### Урок 1.1 — Твоя первая программа
**Экраны:** теория → допиши → задача

#### 📖 Теория
Программа — это список команд для компьютера. Он выполняет их сверху вниз, строка за строкой.

Самая знаменитая первая команда — `print`. Она выводит текст на экран:

```python
print("Привет, мир!")
```
```
Привет, мир!
```

Текст обязательно берём в кавычки — так Python понимает, где команда, а где просто слова.

🐍 Пай: «Нажми ▶ Запустить — и ты официально программист!»

#### ✏️ Допиши код
**Условие:** допиши команду, чтобы программа поздоровалась.
**Стартовый код:**
```python
___("Привет, мир!")
```
**Подсказки:**
1. Команда вывода на экран — её название только что было в теории.
2. Она начинается на «p» и состоит из 5 букв.
3. Решение: `print("Привет, мир!")` — `print` выводит на экран всё, что в скобках.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Привет, мир!` |

#### 💻 Задача
**Условие:** выведи на экран своё любимое блюдо. Например: `Пицца`.
**Стартовый код:** пусто.
**Подсказки:**
1. Используй `print`, не забудь кавычки и скобки.
2. Шаблон: `print("твой текст")`.
3. Решение: `print("Пицца")` — внутри кавычек может быть любой текст.
**Автотесты:** проверка: вывод — одна непустая строка (точное содержимое не сравнивается).

🎉 После запуска: бейдж **«Hello, World!»**

### Урок 1.2 — Кавычки
**Экраны:** теория → квиз → баг

#### 📖 Теория
Кавычки бывают двойные `"..."` и одинарные `'...'` — Python принимает обе, главное чтобы открывающая и закрывающая совпадали:

```python
print("Так можно")
print('И так можно')
```

А вот так нельзя — Python запутается:

```python
print("Так нельзя')
```
```
SyntaxError: unterminated string literal
```

Это твоя первая ошибка — `SyntaxError` значит «не понял, что ты написал». Не страшно: ошибки — это подсказки, а не наказание.

#### ❓ Квиз
Какая строка написана правильно?
- `print('Привет")` — ❌ кавычки разные: открыли одинарной, закрыли двойной.
- ✅ `print('Привет')`
- `print(Привет)` — ❌ без кавычек Python подумает, что `Привет` — это команда, и не найдёт её.
- `print "Привет"` — ❌ потерялись скобки: `print` всегда вызывается со скобками.

#### 🐞 Найди баг
**Код:**
```python
print("Я учу Python!')
```
**Ошибка:** строка 1 — открыта двойная кавычка, закрыта одинарная.
**Правильно:** `print("Я учу Python!")`
**Объяснение после решения:** кавычки — как скобки: какой открыл, такой и закрывай.

### Урок 1.3 — Несколько команд
**Экраны:** теория → допиши → задача

#### 📖 Теория
Программа может состоять из многих команд. Каждый `print` выводит текст **с новой строки**:

```python
print("Раз")
print("Два")
print("Три")
```
```
Раз
Два
Три
```

Порядок важен: Python выполняет строки сверху вниз, как рецепт.

#### ✏️ Допиши код
**Условие:** программа должна вывести список покупок: `Хлеб`, потом `Молоко`, потом `Сыр`. Допиши недостающую строку.
**Стартовый код:**
```python
print("Хлеб")
___
print("Сыр")
```
**Подсказки:**
1. Между хлебом и сыром должно появиться молоко.
2. Нужна ещё одна команда `print` — такая же, как соседние.
3. Решение: `print("Молоко")` — каждая строка вывода = отдельный `print`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Хлеб` ⏎ `Молоко` ⏎ `Сыр` |

#### 💻 Задача
**Условие:** выведи план идеального дня — ровно 3 строки: `Проснуться`, `Поесть`, `Кодить`.
**Стартовый код:** пусто.
**Подсказки:**
1. Понадобятся три команды `print`, по одной на строку плана.
2. Порядок строк в коде = порядок строк на экране.
3. Решение: `print("Проснуться")`, `print("Поесть")`, `print("Кодить")` — три отдельных вызова.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Проснуться` ⏎ `Поесть` ⏎ `Кодить` |

### Урок 1.4 — Что выведет код?
**Экраны:** квиз → квиз → пазл

#### ❓ Квиз 1
Что выведет программа?
```python
print("2 + 2")
```
- `4` — ❌ внутри кавычек — просто текст, Python не считает его.
- ✅ `2 + 2`
- `Ошибка` — ❌ код корректный: обычный текст в кавычках.
- Ничего — ❌ `print` всегда что-то выводит.

**Объяснение после ответа:** всё, что в кавычках, — текст «как есть». Считать Python начнёт, когда кавычек нет — об этом в модуле 2!

#### ❓ Квиз 2
Что выведет программа?
```python
print("Старт")
print("Финиш")
print("Середина")
```
- `Старт Финиш Середина` — ❌ каждый `print` печатает с новой строки.
- `Старт` ⏎ `Середина` ⏎ `Финиш` — ❌ Python не сортирует строки по смыслу — только порядок в коде.
- ✅ `Старт` ⏎ `Финиш` ⏎ `Середина`
- `Ошибка` — ❌ код корректный.

#### 🧩 Собери код
**Условие:** собери программу, которая выводит обратный отсчёт: `3`, `2`, `1`, `Пуск!`
**Строки для перетаскивания (перемешаны):** `print("1")`, `print("Пуск!")`, `print("3")`, `print("2")`
**Правильный порядок:** `print("3")` → `print("2")` → `print("1")` → `print("Пуск!")`

### Урок 1.5 — Комментарии
**Экраны:** теория → допиши → квиз

#### 📖 Теория
Иногда хочется оставить в коде заметку для себя. Для этого есть комментарии — строки, начинающиеся с `#`. Python их полностью игнорирует:

```python
# Это моя первая программа
print("Работает!")  # а это комментарий после кода
```
```
Работает!
```

Комментарии — это записки на полях: компьютеру всё равно, а человеку (особенно тебе-через-месяц) — очень полезно.

#### ✏️ Допиши код
**Условие:** программа выводит лишнюю строку. Преврати её в комментарий, ничего не удаляя.
**Стартовый код:**
```python
print("Нужная строка")
print("Черновик, не выводить")
```
**Подсказки:**
1. Python игнорирует строки, которые начинаются с определённого символа.
2. Поставь `#` в начало второй строки.
3. Решение: `# print("Черновик, не выводить")` — закомментированный код не выполняется. Программисты так «выключают» строки постоянно.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Нужная строка` |

#### ❓ Квиз
Что выведет программа?
```python
# print("Один")
print("Два")  # print("Три")
```
- `Один` ⏎ `Два` — ❌ первая строка закомментирована целиком.
- ✅ `Два`
- `Два` ⏎ `Три` — ❌ всё после `#` на строке — комментарий, включая второй print.
- Ничего — ❌ `print("Два")` стоит до символа `#`, он выполнится.

### Урок 1.6 — print с запятыми
**Экраны:** теория → допиши → задача

#### 📖 Теория
`print` умеет выводить несколько вещей за раз — перечисли их через запятую, и он сам поставит пробелы:

```python
print("Меня", "зовут", "Пай")
```
```
Меня зовут Пай
```

Это удобнее, чем склеивать всё в одну длинную строку.

#### ✏️ Допиши код
**Условие:** допиши вызов так, чтобы вывелось `Питон это весело` — одним print.
**Стартовый код:**
```python
print("Питон", ___, "весело")
```
**Подсказки:**
1. На месте пропуска — ещё один кусочек текста в кавычках.
2. Не хватает слова `"это"`.
3. Решение: `print("Питон", "это", "весело")` — запятая сама добавляет пробел между словами.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Питон это весело` |

#### 💻 Задача
**Условие:** выведи одной командой `print` (с запятыми) строку: `Пицца стоит 599 рублей`. Число `599` напиши без кавычек — числа в кавычках не нуждаются!
**Стартовый код:** пусто.
**Подсказки:**
1. Перечисли через запятую: текст, число, текст.
2. Шаблон: `print("...", 599, "...")`.
3. Решение: `print("Пицца стоит", 599, "рублей")` — числа Python выводит и без кавычек, это пригодится в модуле 2.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Пицца стоит 599 рублей` |

### Урок 1.7 — Пустые строки и \n
**Экраны:** теория → допиши → квиз

#### 📖 Теория
`print()` без всего внутри выводит пустую строку — удобно разделять текст:

```python
print("Глава 1")
print()
print("Глава 2")
```

А ещё есть волшебная пара символов `\n` — «перевод строки» прямо внутри текста:

```python
print("Раз\nДва")
```
```
Раз
Два
```

Один `print` — а строк две!

#### ✏️ Допиши код
**Условие:** выведи `Верх` и `Низ` на разных строках, используя ОДИН print и `\n`.
**Стартовый код:**
```python
print("Верх___Низ")
```
**Подсказки:**
1. Внутри текста есть спецсимвол, который переносит строку.
2. Вставь `\n` между словами.
3. Решение: `print("Верх\nНиз")` — `\n` невидим сам по себе, но разрезает строку на две.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Верх` ⏎ `Низ` |

#### ❓ Квиз
Сколько строк выведет `print("a\nb\nc")`?
- 1 — ❌ каждый `\n` начинает новую строку.
- 2 — ❌ посчитай: `a`, перенос, `b`, перенос, `c`.
- ✅ 3
- Будет ошибка — ❌ `\n` — законная часть строки.

### Урок 1.8 — Охота на баги
**Экраны:** баг → баг → пазл

#### 🐞 Найди баг 1
**Код:**
```python
print("Утро")
primt("Вечер")
```
**Ошибка:** строка 2 — опечатка `primt`.
**Вывод Python:** `NameError: name 'primt' is not defined. Did you mean: 'print'?`
**Объяснение:** Python ищет команду с точно таким именем — и не находит. Кстати, он даже сам предложил исправление: читай ошибки, они дружелюбнее, чем кажутся!

#### 🐞 Найди баг 2
**Код:**
```python
print("Один")
print("Два"
print("Три")
```
**Ошибка:** строка 2 — не закрыта скобка.
**Вывод Python:** `SyntaxError: '(' was never closed`
**Объяснение:** каждая открытая скобка должна быть закрыта. Python заметил это только на следующей строке — поэтому иногда ошибка «показывает» чуть ниже того места, где она на самом деле.

#### 🧩 Собери код
**Условие:** собери программу-визитку. Вывод:
```
Аня
Программист
```
**Строки (перемешаны):** `print("Программист")`, `# Моя визитка`, `print("Аня")`
**Правильный порядок:** `# Моя визитка` → `print("Аня")` → `print("Программист")`
**Примечание для платформы:** комментарий принимается на любой позиции — главное, чтобы print'ы шли в нужном порядке (это честно: комментарий не влияет на вывод).

### Урок 1.10 — 🏆 БОСС: ASCII-открытка
**Экраны:** босс (50 XP)

#### 📖 Вступление
🐍 Пай: «Время первого большого дела! Сделаем открытку из символов — как программисты в 80-х».

#### 💻 Задание
**Условие:** выведи открытку — ровно такую:
```
*****************
*               *
*  С ДНЁМ КОДА! *
*               *
*****************
```
Требования:
1. Минимум 5 команд `print` (или меньше, если используешь `\n` — твой выбор).
2. В начале программы — комментарий с твоим именем.

**Подсказки:**
1. Каждая строка открытки — отдельный print. Скопируй рамку посимвольно.

3. Решение:
```python
# Открытка от Ани
print("*****************")
print("*               *")
print("*  С ДНЁМ КОДА! *")
print("*               *")
print("*****************")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | 5 строк открытки, посимвольное сравнение |
