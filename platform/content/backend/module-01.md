### Урок 1.1 — Клиент и сервер
**Экраны:** теория → квиз → пазл

#### 📖 Теория
Интернет — это разговор двух программ. Одна **просит**, другая **отвечает**.

- **Клиент** — тот, кто просит. Браузер, мобильное приложение, твой Python-скрипт.
- **Сервер** — тот, кто отвечает. Программа, которая всё время ждёт запросы и выдаёт данные.

Жизненная аналогия: ты в кафе. Ты (клиент) говоришь официанту заказ, кухня (сервер) готовит и приносит. Ты не заходишь на кухню — общаешься через понятные правила.

```text
Клиент  →  запрос (request)   →  Сервер
Клиент  ←  ответ  (response)  ←  Сервер
```

Backend-разработчик пишет **серверную** часть: то, что принимает запросы и решает, что ответить.

🐍 Пай: «Запомни одно: клиент просит, сервер отвечает. Всё остальное — детали этого разговора».

#### ❓ Квиз
Ты открыл приложение банка на телефоне, и оно показало баланс. Кто здесь клиент?
- ✅ `Приложение на телефоне` — оно отправило запрос и получило ответ.
- `Сервер банка` — он как раз отвечает, а не просит.
- `Сотрудник банка` — людей в HTTP-разговоре нет, общаются программы.
- `Интернет-провайдер` — он только передаёт данные, но не он автор запроса.

#### 🧩 Собери код
Расставь шаги одного обращения к серверу по порядку.
**Строки:** `Клиент формирует запрос` · `Запрос идёт к серверу` · `Сервер обрабатывает запрос` · `Сервер отправляет ответ` · `Клиент показывает результат`
**Правильный порядок:** `Клиент формирует запрос` · `Запрос идёт к серверу` · `Сервер обрабатывает запрос` · `Сервер отправляет ответ` · `Клиент показывает результат`

### Урок 1.2 — Что такое HTTP
**Экраны:** теория → квиз

#### 📖 Теория
**HTTP** — это язык, на котором клиент и сервер договариваются. Набор правил: как оформить просьбу и как оформить ответ, чтобы обе стороны поняли друг друга.

Каждое обращение состоит из двух частей:
- **Request** (запрос) — просьба клиента: «дай список товаров», «создай платёж».
- **Response** (ответ) — ответ сервера: данные + отметка, всё ли прошло хорошо.

Адрес, по которому стучится клиент, выглядит так:

```text
https://api.shop.com/products?page=2
└─┬─┘   └────┬─────┘ └──┬───┘ └──┬──┘
протокол    хост      путь    параметры
```

HTTPS — это тот же HTTP, только зашифрованный. В реальной работе всегда HTTPS.

🐍 Пай: «HTTP — это не интернет целиком, а именно правила вежливого разговора между программами».

#### ❓ Квиз
Что такое HTTP простыми словами?
- ✅ `Правила, по которым клиент и сервер обмениваются запросами и ответами`
- `Язык программирования для серверов` — нет, это протокол общения, а не язык кода.
- `База данных в интернете` — нет, данные хранит БД, HTTP только передаёт.
- `Программа-браузер` — браузер лишь использует HTTP, но это не одно и то же.

### Урок 1.3 — HTTP-методы
**Экраны:** теория → квиз → задача

#### 📖 Теория
В каждом запросе клиент указывает **метод** — что именно он хочет сделать с данными:

| Метод | Что делает | Пример из жизни |
|-------|-----------|-----------------|
| `GET` | получить данные | посмотреть список товаров |
| `POST` | создать новое | оформить заказ |
| `PUT` | заменить целиком | переписать карточку товара заново |
| `PATCH` | изменить частично | поменять только цену |
| `DELETE` | удалить | отменить заказ |

Главное правило: `GET` ничего не меняет на сервере — только читает. `POST`, `PUT`, `PATCH`, `DELETE` — меняют.

**Ошибка новичка:** отправлять изменение данных через `GET` («чтобы было проще»). Так делать нельзя: браузеры и прокси кэшируют GET-запросы, и операция может повториться сама собой.

🐍 Пай: «GET — посмотреть. POST — создать. DELETE — удалить. Метод сразу говорит о намерении».

#### ❓ Квиз
Пользователь меняет в профиле только номер телефона. Какой метод подходит лучше всего?
- ✅ `PATCH` — меняем часть данных, остальное не трогаем.
- `GET` — он только читает и ничего не меняет.
- `DELETE` — это удаление, а мы обновляем.
- `POST` — обычно для создания нового объекта, а профиль уже есть.

#### 💻 Задача
**Условие:** напиши помощника, который по названию действия подсказывает HTTP-метод. Считай одну строку: `read`, `create`, `update` или `delete` — и выведи метод: `GET`, `POST`, `PATCH` или `DELETE`.
**Стартовый код:**
```python
action = input()
# выведи подходящий метод
```
**Подсказки:**
1. Удобно завести словарь `{"read": "GET", ...}`.
2. Достань значение по ключу: `methods[action]`.
3. Решение:
```python
action = input()
methods = {"read": "GET", "create": "POST", "update": "PATCH", "delete": "DELETE"}
print(methods[action])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `read` | `GET` |
| `create` | `POST` |
| `delete` | `DELETE` |

### Урок 1.4 — Статус-коды
**Экраны:** теория → задача → квиз

#### 📖 Теория
В каждом ответе сервер ставит **статус-код** — короткий итог: получилось или нет.

- **2xx — успех.** `200 OK`, `201 Created` (что-то создано).
- **4xx — ошибка клиента.** Ты сам что-то прислал не так: `400` (плохой запрос), `401` (не авторизован), `403` (нет прав), `404` (не найдено), `422` (данные не прошли проверку).
- **5xx — ошибка сервера.** Сломалось на той стороне: `500` (внутренняя ошибка).

Аналогия: 2xx — «готово!», 4xx — «ты перепутал заказ», 5xx — «у нас на кухне пожар».

**Ошибка новичка:** видеть `404` и думать «интернет сломался». На самом деле сервер ответил нормально — он просто говорит «такого ресурса нет».

🐍 Пай: «Первая цифра кода — это эмоция ответа. 2 — радость, 4 — твоя ошибка, 5 — их ошибка».

#### 💻 Задача
**Условие:** напиши «переводчик» статус-кодов. Считай число и выведи короткое значение: `200`→`OK`, `201`→`Created`, `400`→`Bad Request`, `401`→`Unauthorized`, `404`→`Not Found`, `500`→`Server Error`. Для незнакомого кода выведи `Unknown`.
**Стартовый код:**
```python
code = int(input())
# выведи значение кода
```
**Подсказки:**
1. Снова словарь: ключ — число, значение — текст.
2. Для незнакомого кода удобен `.get(code, "Unknown")`.
3. Решение:
```python
code = int(input())
names = {200: "OK", 201: "Created", 400: "Bad Request",
         401: "Unauthorized", 404: "Not Found", 500: "Server Error"}
print(names.get(code, "Unknown"))
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `200` | `OK` |
| `404` | `Not Found` |
| `401` | `Unauthorized` |
| `418` | `Unknown` |

#### ❓ Квиз
Сервер вернул `401`. Что это значит?
- ✅ `Пользователь не авторизован или токен неправильный`
- `Страница не найдена` — это `404`, а не `401`.
- `Сервер сломался` — это `500`, ошибка на стороне сервера.
- `Всё хорошо, данные получены` — успех это `200`, а `401` — ошибка клиента.

### Урок 1.5 — Headers, body и параметры
**Экраны:** теория → задача → баг

#### 📖 Теория
Запрос — это не только метод и адрес. У него есть ещё три важные части.

- **Headers (заголовки)** — служебные пометки: кто я, какой формат жду, мой токен. Пример: `Authorization: Bearer xxx`, `Content-Type: application/json`.
- **Body (тело)** — основное содержимое. В `POST`-запросе тут едут данные нового объекта (обычно JSON).
- **Параметры** — уточнения прямо в адресе:
  - **query** — после `?`: `/products?page=2&sort=price` (фильтры, страницы);
  - **path** — внутри пути: `/products/42` (число 42 — это id товара).

Аналогия: headers — это конверт (от кого, куда), body — письмо внутри, query/path — точный адрес на конверте.

🐍 Пай: «Path-параметр отвечает на вопрос „какой именно объект“, query-параметр — „как его показать“».

#### 💻 Задача
**Условие:** клиент прислал строку query-параметров вида `page=2&sort=price`. Считай её и выведи значение параметра `page`.
**Стартовый код:**
```python
query = input()  # например: page=2&sort=price
# выведи значение page
```
**Подсказки:**
1. Сначала раздели строку по `&`, потом каждую часть — по `=`.
2. Собери словарь: `dict(part.split("=") for part in query.split("&"))`.
3. Решение:
```python
query = input()
params = dict(part.split("=") for part in query.split("&"))
print(params["page"])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `page=2&sort=price` | `2` |
| `sort=name&page=5` | `5` |
| `page=1` | `1` |

#### 🐞 Найди баг
**Код:**
```python
# хотим достать id товара из пути /products/42
path = "/products/42"
product_id = path[0]
print(product_id)
```
**Ошибка:** строка 3 — `path[0]` берёт первый символ `/`, а не число после последнего `/`.
**Правильно:** `product_id = path.split("/")[-1]`
**Объяснение после решения:** path-параметр — это часть пути; чтобы вытащить последний сегмент, режем строку по `/` и берём последний кусок `[-1]`.

### Урок 1.6 — 🏆 БОСС: разбор запроса
**Экраны:** теория → задача → квиз

#### 📖 Теория
Собери всё вместе. Любой HTTP-запрос можно прочитать по частям:

```text
POST /payments?notify=true        ← метод + путь + query
Authorization: Bearer abc123       ← заголовок
Content-Type: application/json      ← заголовок
                                    ← пустая строка
{"amount": 500000, "currency":"UZS"} ← тело (body)
```

Сервер прочитает это так: «клиент с токеном abc123 хочет **создать** платёж, просит уведомить (`notify=true`), данные платежа — в теле».

🐍 Пай: «Прочитал запрос по строчкам — считай, уже наполовину backend-разработчик».

#### 💻 Задача
**Условие:** тебе дают первую строку HTTP-запроса вида `POST /payments?notify=true`. Выведи метод и путь без query — через пробел. Для примера выше: `POST /payments`.
**Стартовый код:**
```python
line = input()  # например: POST /payments?notify=true
# выведи: метод и путь без query
```
**Подсказки:**
1. Раздели строку по пробелу: `method, target = line.split(" ")`.
2. Путь без query — это часть до `?`: `target.split("?")[0]`.
3. Решение:
```python
line = input()
method, target = line.split(" ")
path = target.split("?")[0]
print(method, path)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `POST /payments?notify=true` | `POST /payments` |
| `GET /products` | `GET /products` |
| `DELETE /orders/7?force=1` | `DELETE /orders/7` |

#### ❓ Квиз
В запросе `GET /products/42?currency=USD` что такое `42`?
- ✅ `Path-параметр — id конкретного товара`
- `Query-параметр` — query идёт после `?`, это `currency=USD`.
- `Заголовок` — заголовки вообще не в строке адреса.
- `Тело запроса` — у GET тела обычно нет, и тело это не часть пути.
