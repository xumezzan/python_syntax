# Трек «Python Backend: API, базы данных и интеграции» — структура

18 модулей, 128 уроков. Каждый модуль — отдельный файл `module-XX.md`, формат
тот же, что в синтаксис-курсе (см. `../00-структура-курса.md`). Глобальные id
модулей сквозные (11–28), в интерфейсе показываются как «Модуль 1–18».

## Особенность авто-проверки в этом треке

Код выполняется в браузере через Pyodide (CPython→WASM). Доступна вся стандартная
библиотека (`json`, `sqlite3`, `xml.etree`, `base64`, `hashlib`), но **нет**
`fastapi`, `sqlalchemy`, `httpx` и сети. Поэтому:

- **авто-тестируемые задачи** написаны на чистом Python: разбор JSON/XML, SQL на
  sqlite, логика retry/idempotency, маппинг данных, статус-коды — они реально
  запускаются и сверяются с эталоном (102 тест-кейса);
- **задачи по FastAPI / SQLAlchemy / Docker / Git** — в режиме самопроверки
  (`**Автотесты:** самопроверка: …`) и через квизы, «найди баг», «собери код»;
- запуск настоящего FastAPI-стека — в финальном проекте `enterprise-integration/`.

## Как добавить/изменить контент

Контент — это markdown. Правишь `module-XX.md` — урок обновляется без пересборки.
Минимумы на модуль: ≥3 урока, ≥2 практических задания (код/баг/пазл), ≥3 квиза,
чеклист модуля. Проверка: `npm run check`. Прогон эталонных решений авто-задач:
`node scripts/verify-solutions.mjs`.
