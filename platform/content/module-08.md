### Урок 8.1 — def: своя команда
**Экраны:** теория → допиши → задача

#### 📖 Теория
Ты постоянно пользуешься чужими командами: `print`, `len`, `input`. Пора создавать свои! Функция — это кусок кода с именем:

```python
def greet():
    print("Привет!")
    print("Рад тебя видеть")

greet()     # вызов — код выполняется
greet()     # и ещё раз — без копипасты!
```

`def`, имя, скобки, двоеточие — и тело с отступом. Важно: при объявлении функция НЕ выполняется. Она ждёт вызова — как рецепт ждёт повара.

#### ✏️ Допиши код
**Условие:** объяви функцию-сирену и вызови её дважды.
**Стартовый код:**
```python
___ alarm():
    print("ТРЕВОГА!")

alarm()
alarm()
```
**Подсказки:**
1. Ключевое слово для объявления функции — 3 буквы.
2. `def`.
3. Решение: `def alarm():`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `ТРЕВОГА!` ⏎ `ТРЕВОГА!` |

#### 💻 Задача
**Условие:** напиши функцию `line()`, которая печатает 20 дефисов, и используй её, чтобы вывести меню:
```
--------------------
МЕНЮ
--------------------
```
**Подсказки:**
1. Внутри функции — `print("-" * 20)`.
2. Вызовы: line(), print("МЕНЮ"), line().
3. Решение:
```python
def line():
    print("-" * 20)

line()
print("МЕНЮ")
line()
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|

### Урок 8.2 — Параметры
**Экраны:** теория → допиши → задача

#### 📖 Теория
Функции можно передавать данные — через параметры в скобках:

```python
def greet(name):
    print(f"Привет, {name}!")

greet("Аня")     # Привет, Аня!
greet("Пай")     # Привет, Пай!
```

`name` — параметр: переменная-приёмник внутри функции. `"Аня"` — аргумент: конкретное значение при вызове. Одна функция — разные результаты.

#### ✏️ Допиши код
**Условие:** функция должна принимать имя блюда.
**Стартовый код:**
```python
def cook(___):
    print(f"Готовлю {dish}...")
    print(f"{dish} готово!")

cook("борщ")
cook("суши")
```
**Подсказки:**
1. Параметр пишется в скобках после имени функции.
2. Какое имя используется внутри? `dish`.
3. Решение: `def cook(dish):`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Готовлю борщ...` ⏎ `борщ готово!` ⏎ `Готовлю суши...` ⏎ `суши готово!` |

#### 💻 Задача
**Условие:** напиши функцию `repeat(word, times)`, печатающую слово указанное число раз (каждое на своей строке). Вызови `repeat("эхо", 3)`.
**Подсказки:**
1. Два параметра через запятую, внутри — цикл for.
2. `for i in range(times): print(word)`.
3. Решение:
```python
def repeat(word, times):
    for i in range(times):
        print(word)

repeat("эхо", 3)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `эхо` ⏎ `эхо` ⏎ `эхо` |

### Урок 8.3 — return: функция отвечает
**Экраны:** теория → квиз → задача

#### 📖 Теория
Самое важное в модуле. Функция может не печатать, а **возвращать** результат:

```python
def double(x):
    return x * 2

a = double(5)        # a = 10 — результат можно сохранить
print(double(a))     # 20 — или передать дальше
print(double(3) + double(4))   # 14 — или посчитать с ним!
```

Разница между print и return:
- `print` — показать человеку. Программа результат не видит.
- `return` — отдать программе. Дальше с ним можно работать.

Функция с print — тупик. Функция с return — конвейер. Хорошие функции почти всегда возвращают.

#### ❓ Квиз
Что выведет код?
```python
def add(a, b):
    return a + b

result = add(2, 3)
print(result * 10)
```
- `5` — ❌ результат ещё умножили на 10.
- ✅ `50`
- `2310` — ❌ a и b — числа, плюс их сложил, а не склеил.
- Ничего — ❌ print в последней строке есть.

#### 💻 Задача
**Условие:** напиши функцию `to_rub(usd)`, переводящую доллары в рубли по курсу 90, — она ВОЗВРАЩАЕТ результат. Потом прочитай число и выведи `f"{usd}$ = {to_rub(usd)} руб."`.
**Подсказки:**
1. В функции одна строка: `return usd * 90`.
2. Вызов функции можно писать прямо внутри f-строки.
3. Решение:
```python
def to_rub(usd):
    return usd * 90

usd = int(input())
print(f"{usd}$ = {to_rub(usd)} руб.")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `10` | `10$ = 900 руб.` |
| `0` | `0$ = 0 руб.` |

### Урок 8.4 — return завершает функцию
**Экраны:** теория → квиз → задача

#### 📖 Теория
`return` — это ещё и выход: функция заканчивается немедленно:

```python
def check_age(age):
    if age < 0:
        return "Это шутка?"
    if age < 18:
        return "Несовершеннолетний"
    return "Взрослый"
```

Дошли до первого подходящего return — всё, выход. Поэтому else здесь даже не нужен: если функция дожила до последней строки, значит, age >= 18. Этот стиль называют «ранний выход» — плоско и читаемо.

Код после return — мёртвый, он не выполнится никогда.

#### ❓ Квиз
Что выведет код?
```python
def magic(x):
    return x + 1
    print("Я выполнюсь?")

print(magic(5))
```
- `6` ⏎ `Я выполнюсь?` — ❌ return завершил функцию: print внутри — мёртвый код.
- ✅ `6`
- `Я выполнюсь?` ⏎ `6` — ❌ до print управление не доходит.
- `Ошибка` — ❌ мёртвый код — плохо, но не ошибка.

#### 💻 Задача
**Условие:** напиши функцию `grade(score)` (балл 0–100): 90+ → `A`, 70+ → `B`, 50+ → `C`, иначе `F`. Используй ранний выход без else. Прочитай число и выведи результат функции.
**Подсказки:**
1. Цепочка if + return, от большего порога к меньшему.
2. Последняя строка функции — просто `return "F"`.
3. Решение:
```python
def grade(score):
    if score >= 90:
        return "A"
    if score >= 70:
        return "B"
    if score >= 50:
        return "C"
    return "F"

print(grade(int(input())))
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `95` | `A` |
| `70` | `B` |
| `49` | `F` |

### Урок 8.5 — Значения по умолчанию
**Экраны:** теория → допиши → квиз

#### 📖 Теория
Параметру можно задать запасное значение — оно сработает, если аргумент не передали:

```python
def order_pizza(size="средняя", cheese=True):
    print(f"Пицца: {size}, доп. сыр: {cheese}")

order_pizza()                      # Пицца: средняя, доп. сыр: True
order_pizza("большая")             # Пицца: большая, доп. сыр: True
order_pizza(cheese=False)          # Пицца: средняя, доп. сыр: False
```

Последний вызов — **именованный аргумент**: говоришь, КАКОЙ параметр задаёшь, и пропускаешь остальные. Ты такое уже видел: `sort(reverse=True)`, `print(end="")` — теперь понятно, как они устроены!

**Подсказки:**

### Урок 8.6 — Вернуть несколько значений
**Экраны:** теория → задача

#### 📖 Теория
Функция может вернуть сразу несколько значений — это кортеж + распаковка из модуля 6:

```python
def min_max(nums):
    return min(nums), max(nums)

low, high = min_max([3, 1, 4, 1, 5])
print(low, high)     # 1 5
```

Возвращаешь через запятую — принимаешь через запятую. Никаких костылей, это идиоматичный Python.

#### 💻 Задача
**Условие:** напиши функцию `split_bill(total, people)`, возвращающую два значения: сколько платит каждый (целая часть) и сколько рублей не хватает (остаток). Прочитай два числа и выведи `По 333 с носа, не хватает 1` (для 1000 и 3).
**Подсказки:**
1. Это `//` и `%` из модуля 2.
2. `return total // people, total % people`.

### Урок 8.7 — Области видимости
**Экраны:** теория → квиз → баг

#### 📖 Теория
Переменные, созданные внутри функции, живут только внутри неё:

```python
def mix():
    secret = "тссс"
    print(secret)     # работает

mix()
print(secret)         # NameError! снаружи её нет
```

Это **локальная** переменная. И наоборот: функция видит внешние (глобальные) переменные, но менять их без спроса не может — при присваивании просто создаст свою локальную копию.

Звучит строго, но это защита: функции не топчутся по чужим переменным. Золотое правило: всё, что нужно функции, — передавай через параметры; всё, что она наработала, — забирай через return.

#### ❓ Квиз
Что выведет код?
```python
x = 10

def change():
    x = 99

change()
print(x)
```
- `99` — ❌ присваивание внутри функции создало ЛОКАЛЬНУЮ x, внешняя не тронута.
- ✅ `10`
- `Ошибка` — ❌ кода корректный, в этом и подвох.
- `99` ⏎ `10` — ❌ внутри функции ничего не печатается.

#### 🐞 Найди баг
**Код:**
```python
def get_discount(price):
    discount = price // 10

result = get_discount(500)
print(result)
```
**Задумка:** вывести 50.
**Ошибка:** функция ничего не возвращает — `result` равен `None`.
**Правильно:** `return price // 10` вместо строки с discount (или после неё).
**Объяснение:** функция без return возвращает None — невидимое «ничего». Посчитал — верни!

### Урок 8.8 — Функции зовут функции
**Экраны:** теория → задача

#### 📖 Теория
Настоящие программы — это функции, собранные из функций:

```python
def clean(text):
    return text.strip().lower()

def is_yes(answer):
    return clean(answer) == "да"

if is_yes(input()):
    print("Поехали!")
```

Каждая функция делает ОДНУ маленькую вещь и делает её хорошо. Большая задача рассыпается на простые кубики. Это и есть главный навык программиста — декомпозиция.

#### 💻 Задача
**Условие:** собери конвейер из двух функций:
- `count_vowels(word)` — возвращает число гласных (`аеёиоуыэюя`) в слове;
- `describe(word)` — возвращает строку `слово: N гласных`, используя первую функцию.
Прочитай слово и выведи результат `describe`.
**Подсказки:**
1. В первой — счётчик и `for ch in word: if ch in "аеёиоуыэюя"`.
2. Во второй — f-строка с вызовом первой: `f"{word}: {count_vowels(word)} гласных"`.
3. Решение:
```python
def count_vowels(word):
    count = 0
    for ch in word:
        if ch in "аеёиоуыэюя":
            count += 1
    return count

def describe(word):
    return f"{word}: {count_vowels(word)} гласных"

print(describe(input()))
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `молоко` | `молоко: 3 гласных` |
| `вскрик` | `вскрик: 1 гласных` |

### Урок 8.9 — Документация и понятные имена
**Экраны:** теория → квиз → пазл

#### 📖 Теория
Хорошую функцию понятно, как использовать, не читая её код. Для этого:

1. **Имя-глагол**, говорящее, что она делает: `count_words`, `send_email`, `is_empty` (для True/False — приставка is/has).
2. **Docstring** — описание в тройных кавычках первой строкой:

```python
def final_price(price, discount=0):
    """Возвращает цену с учётом скидки в процентах."""
    return price * (100 - discount) / 100
```

Docstring показывается во всплывающей подсказке редактора — да-да, те подсказки, что ты видишь у print и len, написаны точно так же.

#### ❓ Квиз
Какое имя лучше для функции, проверяющей, пуст ли список?
- `check()` — ❌ что check? чего check? Имя-загадка.
- `empty_list_checker_function()` — ❌ длинно и всё равно неясно, что вернёт.
- ✅ `is_empty(items)`
- `f1(x)` — ❌ через неделю сам не вспомнишь, что это.

#### 🧩 Собери код
**Условие:** собери функцию с docstring в правильном порядке.
**Строки (перемешаны):** `    return price * 2`, `def double_price(price):`, `    """Возвращает удвоенную цену."""`
**Правильный порядок:** `def double_price(price):` → `    """Возвращает удвоенную цену."""` → `    return price * 2`

### Урок 8.10 — Марафон повторения
**Экраны:** квиз → баг → задача

#### ❓ Квиз
Что выведет код?
```python
def shout(text):
    return text.upper() + "!"

print(shout("ура"))
print(shout(shout("ой")))
```
- `УРА!` ⏎ `ОЙ!` — ❌ во втором случае shout применился дважды.
- ✅ `УРА!` ⏎ `ОЙ!!`
- `УРА!` ⏎ `ОЙ!!!` — ❌ восклицательных два: по одному с каждого вызова.
- `Ошибка` — ❌ результат функции можно передать в неё же.

#### 🐞 Найди баг
**Код:**
```python
def get_total(prices):
    total = sum(prices)
    return total

cart = [100, 200, 300]
print(get_total)
```
**Ошибка:** строка 6 — функция не ВЫЗВАНА (нет скобок и аргумента): печатается `<function get_total at 0x...>`.
**Правильно:** `print(get_total(cart))`.
**Объяснение:** имя функции без скобок — это сама функция как объект, а не её результат.

#### 💻 Задача

### Урок 8.11 — 🏆 БОСС: Мини-библиотека
**Экраны:** босс (50 XP)

#### 📖 Вступление
🐍 Пай: «Программисты собирают свои наборы инструментов годами. Начни свой — напиши библиотеку из трёх функций и программу-демо».

#### 💻 Задание
**Условие:** напиши три функции (каждая с docstring!):
1. `normalize(text)` — возвращает текст без пробелов по краям, с большой буквы: `"  аня  "` → `"Аня"` (методы strip и capitalize);
2. `stars(n)` — возвращает строку из n звёздочек: `stars(5)` → `"*****"`;
3. `rate_word(word)` — возвращает оценку слова по длине: длина < 4 → 1, < 7 → 2, иначе 3.

Основная программа: прочитай слово, нормализуй его и выведи две строки: нормализованное слово и звёздочки по его рейтингу.

**Пример:** ввод `  питонище  ` → вывод `Питонище` ⏎ `***`

**Подсказки:**
1. Три def сверху, основная программа снизу. Каждую функцию проверяй отдельно, прежде чем писать следующую.
2. `capitalize()` — метод строки: первая буква большая, остальные маленькие. Рейтинг — ранний выход из урока 8.4.
3. Решение:
```python
def normalize(text):
    """Убирает пробелы по краям, делает первую букву заглавной."""
    return text.strip().capitalize()

def stars(n):
    """Возвращает строку из n звёздочек."""
    return "*" * n

def rate_word(word):
    """Оценивает слово по длине: 1, 2 или 3."""
    if len(word) < 4:
        return 1
    if len(word) < 7:
        return 2
    return 3

word = normalize(input())
print(word)
print(stars(rate_word(word)))
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
