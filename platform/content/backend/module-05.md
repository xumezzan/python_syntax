### Урок 5.1 — Что такое база данных
**Экраны:** теория → квиз

#### 📖 Теория
**База данных (БД)** — это место, где сервер помнит данные между запросами. Закрыл программу, открыл снова — данные на месте.

Самый распространённый вид — **реляционная** БД (PostgreSQL, MySQL). Данные хранятся в **таблицах**, как в Excel:
- **таблица** — один тип сущностей (`users`, `products`);
- **строка (row)** — одна запись (один пользователь);
- **колонка (column)** — одно свойство (`email`, `name`).

```text
таблица users
┌────┬──────────────┬───────┐
│ id │ email        │ role  │   ← колонки
├────┼──────────────┼───────┤
│ 1  │ ivan@mail.uz │ admin │   ← строка
│ 2  │ anna@mail.uz │ user  │
└────┴──────────────┴───────┘
```

🐍 Пай: «Таблица — это лист Excel со строгими правилами: у каждой колонки свой тип, и его нельзя нарушить».

#### ❓ Квиз
Что такое «строка» (row) в таблице базы данных?
- ✅ `Одна запись — например, один конкретный пользователь`
- `Одно свойство, например email` — это колонка, а не строка.
- `Вся таблица целиком` — таблица состоит из многих строк.
- `Тип данных` — тип у колонки, строка же это запись.

### Урок 5.2 — Ключи и связи
**Экраны:** теория → квиз

#### 📖 Теория
- **Primary key (первичный ключ)** — колонка, которая уникально определяет строку. Обычно `id`. Двух строк с одним id быть не может.
- **Foreign key (внешний ключ)** — колонка, которая ссылается на id другой таблицы. Так строки связываются между собой.

```text
orders.user_id  →  users.id
```

Это значит: у заказа есть `user_id`, и он указывает, какому пользователю заказ принадлежит. Связь «один пользователь — много заказов».

Аналогия: primary key — это номер паспорта (уникален у каждого). Foreign key — графа «владелец» в техпаспорте машины, ссылается на конкретный паспорт.

🐍 Пай: «Primary key — кто я. Foreign key — на кого я ссылаюсь. Так таблицы держатся за руки».

#### ❓ Квиз
В таблице `orders` есть колонка `user_id`, ссылающаяся на `users.id`. Как называется такая колонка?
- ✅ `Foreign key — внешний ключ, связь с другой таблицей`
- `Primary key` — первичный ключ уникально определяет строку этой же таблицы.
- `Index` — индекс ускоряет поиск, но это не про связь таблиц.
- `Column type` — это не тип, а роль колонки-ссылки.

### Урок 5.3 — SELECT и WHERE
**Экраны:** теория → задача

#### 📖 Теория
**SELECT** — главный запрос: «дай мне данные».

```sql
SELECT name FROM products;            -- все названия
SELECT * FROM products;               -- все колонки
SELECT name FROM products WHERE price > 100;  -- с условием
```

- после `SELECT` — какие колонки (или `*` — все);
- `FROM` — из какой таблицы;
- `WHERE` — условие отбора.

🐍 Пай: «SELECT — что показать, FROM — откуда, WHERE — какие именно. Три слова — и ты читаешь базу».

#### 💻 Задача
**Условие:** в учебной базе есть таблица `products(id, name, price)`. Допиши SQL-запрос в переменную `query`, чтобы вывести названия товаров дороже 100, по возрастанию id.
**Стартовый код:**
```python
import sqlite3
db = sqlite3.connect(":memory:")
db.executescript('''
CREATE TABLE products (id INTEGER, name TEXT, price INTEGER);
INSERT INTO products VALUES (1, 'Pen', 50), (2, 'Bag', 300), (3, 'Cup', 150);
''')

query = "SELECT name FROM products WHERE ___ ORDER BY id"
for row in db.execute(query):
    print(row[0])
```
**Подсказки:**
1. Условие «дороже 100» — это `price > 100`.
2. Подставь его вместо `___`.
3. Решение:
```python
query = "SELECT name FROM products WHERE price > 100 ORDER BY id"
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Bag` ⏎ `Cup` |

### Урок 5.4 — INSERT, UPDATE, DELETE
**Экраны:** теория → квиз

#### 📖 Теория
Чтение — это `SELECT`. Изменение данных — три другие команды:

```sql
INSERT INTO products (name, price) VALUES ('Cup', 150);  -- добавить
UPDATE products SET price = 200 WHERE id = 3;            -- изменить
DELETE FROM products WHERE id = 3;                       -- удалить
```

**Ошибка новичка:** забыть `WHERE` в `UPDATE` или `DELETE`. Без условия команда применится **ко всем строкам** — можно случайно обнулить всю таблицу. Всегда уточняй, какие строки трогаешь.

🐍 Пай: «DELETE без WHERE — это удалить всё. Перед запуском всегда спроси себя: какие строки я выбираю?».

#### ❓ Квиз
Что произойдёт от запроса `UPDATE payments SET status = 'failed';` (без WHERE)?
- ✅ `Статус failed проставится во ВСЕ строки таблицы`
- `Изменится только последняя строка` — без WHERE затрагиваются все строки.
- `Ничего, запрос с ошибкой` — синтаксис верный, поэтому он выполнится.
- `Обновится только первая строка` — нет, без условия — все строки сразу.

### Урок 5.5 — JOIN: связываем таблицы
**Экраны:** теория → задача

#### 📖 Теория
Данные разложены по таблицам, но часто нужно показать их вместе: платёж + имя пользователя. Для этого таблицы **соединяют** по ключу через `JOIN`:

```sql
SELECT users.name, payments.amount
FROM payments
JOIN users ON payments.user_id = users.id;
```

Читается так: «возьми платежи, к каждому подставь пользователя, у которого `users.id` совпадает с `payments.user_id`».

🐍 Пай: «JOIN — это как склеить два листа Excel по общей колонке. ON говорит, по какой именно».

#### 💻 Задача
**Условие:** есть `users(id, name)` и `payments(id, user_id, amount)`. Допиши условие соединения после `ON`, чтобы вывести имя пользователя и сумму каждого платежа (по возрастанию payments.id).
**Стартовый код:**
```python
import sqlite3
db = sqlite3.connect(":memory:")
db.executescript('''
CREATE TABLE users (id INTEGER, name TEXT);
CREATE TABLE payments (id INTEGER, user_id INTEGER, amount INTEGER);
INSERT INTO users VALUES (1, 'Ivan'), (2, 'Anna');
INSERT INTO payments VALUES (1, 1, 500), (2, 2, 300);
''')

query = '''
SELECT users.name, payments.amount
FROM payments
JOIN users ON ___
ORDER BY payments.id
'''
for row in db.execute(query):
    print(row[0], row[1])
```
**Подсказки:**
1. Соединяем по ключу: `payments.user_id` равен `users.id`.
2. Подставь `payments.user_id = users.id` вместо `___`.
3. Решение:
```python
query = '''
SELECT users.name, payments.amount
FROM payments
JOIN users ON payments.user_id = users.id
ORDER BY payments.id
'''
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Ivan 500` ⏎ `Anna 300` |

### Урок 5.6 — Индексы простыми словами
**Экраны:** теория → квиз

#### 📖 Теория
Когда строк миллионы, поиск по `WHERE email = ...` без подготовки идёт по всей таблице — медленно. **Индекс** — это как алфавитный указатель в конце книги: база находит нужную строку сразу, не листая всё.

```sql
CREATE INDEX idx_users_email ON users(email);
```

Индексы ставят на колонки, по которым часто ищут (email, внешние ключи). Минус: индексы занимают место и чуть замедляют запись. Поэтому их добавляют осознанно, а не на всё подряд.

🐍 Пай: «Индекс — указатель в книге. Ищешь по колонке часто — поставь индекс, и поиск станет мгновенным».

#### ❓ Квиз
Зачем нужен индекс на колонке `email`?
- ✅ `Чтобы быстро находить строки по email, не перебирая всю таблицу`
- `Чтобы запретить повторы email` — это делает ограничение UNIQUE, а не сам индекс.
- `Чтобы хранить больше данных` — индекс наоборот занимает дополнительное место.
- `Чтобы зашифровать email` — индекс к шифрованию отношения не имеет.

### Урок 5.7 — 🏆 БОСС: неуспешные платежи
**Экраны:** теория → задача

#### 📖 Теория
Частая задача в платёжных системах: найти все **failed**-платежи и понять, чьи они. Это `JOIN` + `WHERE` вместе.

🐍 Пай: «Связать платежи с людьми и отфильтровать проблемные — типичный отчёт, который backend отдаёт каждый день».

#### 💻 Задача
**Условие:** есть `users(id, name)` и `payments(id, user_id, amount, status)`. Выведи имя владельца и сумму всех платежей со статусом `failed`, по возрастанию payments.id. Допиши запрос.
**Стартовый код:**
```python
import sqlite3
db = sqlite3.connect(":memory:")
db.executescript('''
CREATE TABLE users (id INTEGER, name TEXT);
CREATE TABLE payments (id INTEGER, user_id INTEGER, amount INTEGER, status TEXT);
INSERT INTO users VALUES (1, 'Ivan'), (2, 'Anna');
INSERT INTO payments VALUES
  (1, 1, 500, 'success'),
  (2, 2, 300, 'failed'),
  (3, 1, 700, 'failed');
''')

query = '''
SELECT users.name, payments.amount
FROM payments
JOIN users ON payments.user_id = users.id
WHERE ___
ORDER BY payments.id
'''
for row in db.execute(query):
    print(row[0], row[1])
```
**Подсказки:**
1. Соединение уже готово, осталось условие на статус.
2. `payments.status = 'failed'`.
3. Решение:
```python
query = '''
SELECT users.name, payments.amount
FROM payments
JOIN users ON payments.user_id = users.id
WHERE payments.status = 'failed'
ORDER BY payments.id
'''
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Anna 300` ⏎ `Ivan 700` |
