# PySyntax — дизайн-система и стратегия продукта

Версия 1.0 · Полное переосмысление UI/UX, геймификации и удержания.
Документ привязан к реальному коду: `platform/` (Next.js + Monaco + Pyodide),
контент `content/` (10 модулей, 99 уроков, XP-экономика 10/15/50, маскот Пай,
3 уровня подсказок, боссы, режим «Играть»). Ничего из продуктовой логики не ломаем —
надстраиваем смысл, эмоцию и привычку.

---

## A. Общая дизайн-концепция

### Идея: «Терминал, который стал тёплым»

Код — герой интерфейса. Вся айдентика строится вокруг одной метафоры:
**тёмное окно терминала, которое отвечает тебе по-человечески**. Страх новичка —
это страх холодного чёрного экрана; PySyntax берёт этот экран и делает его
дружелюбным: курсор мигает приветливо, ошибки говорят на русском голосом Пая,
каждый запуск кода — маленький диалог, а не экзамен.

### Ключевой ownable-паттерн: «код живёт в ночи»

Где бы ни был пользователь — в тёмной теме или (в будущем) светлой — **все
кодовые поверхности всегда тёмные**: пример, плейграунд, вывод. Код = ночь =
фокус. Текст и навигация могут быть светлыми, код — никогда. Это единственное
жёсткое правило бренда, и оно мгновенно отличает PySyntax от шаблонных дашбордов.

### Настроение

Спокойная уверенность наставника. Не «детская игра» и не «хакерский кинофильм»:
глубокий сине-графитовый фон, тёплый REPL-зелёный, аккуратное золото для наград.
Премиально, как Linear; тепло, как Duolingo; тихо, как Notion.

### Чем отличается от типичного AI-dashboard

- Нет сеток из одинаковых серых виджетов — есть **один путь** сверху вниз.
- Иконки модулей — не эмодзи и не абстракции, а **глифы из самого кода**
  (см. раздел E): `>` для первых шагов, `=` для переменных, `?` для условий,
  `↻` для циклов, `[ ]` для списков, `ƒ( )` для функций. Python-вайб без змеек и клише.
- Микрокопия от первого лица наставника, а не системные надписи.
- Празднования заслуженные и редкие: конфетти только за первую программу,
  босса и модуль — поэтому они работают.

### Три дизайн-направления (рекомендация — №3)

| № | Направление | Суть | Кому |
|---|------------|------|------|
| 1 | **Night Terminal** | текущая тёмная тема, доведённая до ума: графит + REPL-зелёный, терминальные мотивы | подростки и взрослые, вечерние сессии |
| 2 | **Playground Light** | премиальная светлая: бумага #FAFAF7, чернильный текст, приглушённые candy-акценты | школьники, дневная учёба, «не страшно» |
| 3 | **Day/Night Dual** ★ | светлая оболочка опциональна, но **код всегда тёмный** — терминал-в-странице как фирменный приём | все; масштабируется, узнаваем |

Стартуем с night-first (уже реализовано), закладываем токены под dual.

## B. UX-стратегия

### Путь пользователя (макроуровень)

```
Лендинг-обещание → 90-секундный онбординг с первой программой →
Дневная петля (карточка «Сегодня» → урок 3–5 мин → награда → тизер завтра) →
Недельная петля (квесты → recap → босс недели) →
Вехи (модуль = глава = печать) → Выпуск (сертификат) → следующий курс
```

### Главные принципы

1. **Один следующий шаг.** На любом экране ровно одна главная кнопка.
   Пользователь никогда не выбирает «что учить» — он жмёт «Продолжить».
2. **Эффект незавершённости (Zeigarnik).** Начатое всегда видно: «Урок 2.4 ·
   шаг 2 из 3», модуль на 80% — главный магнит возврата. Прогресс-бары
   показываем всегда, когда они > 0%, и никогда — пустыми и грустными.
3. **Градиент цели.** Чем ближе цель, тем заметнее остаток: «до уровня — 12 XP»,
   «до печати модуля — 2 урока». Близкие цели дожимают сессию.
4. **Быстрая первая победа.** Первая программа — через 60–90 секунд после
   входа, до любой теории. Бейдж «Hello, World!» — вручён, прогресс «надарен»
   (endowed progress: полоска онбординга стартует с 1/5, а не с 0).
5. **Ошибки — норма.** Ошибка не красит экран в красный: вывод Пая жёлтый,
   тёплый, с конкретикой. Кнопка «Проверить» никогда не наказывает — счётчик
   попыток не показываем, показываем только успехи.

### Как уменьшается страх новичка

- Обещание времени рядом с каждым действием: «≈ 3 мин» (implementation intention).
- Первый экран урока — всегда читаемая история, не код.
- Подсказки легальны и встроены: «Подсказка 1 из 3 · бесплатно» — просить
  помощь спроектировано как нормальный шаг, а не поражение.
- Перед сложным (боссы) — явное «у тебя уже есть всё, чтобы это решить:
  циклы ✓ условия ✓ накопитель ✓».

## C. Геймификация

Принцип: **каждая механика отвечает на вопрос пользователя**, а не сыплет очки.

| Вопрос пользователя | Механика | Реализация |
|---|---|---|
| «Расту ли я?» | XP → уровни с титулами | кривая: до уровня N нужно 75+25·N XP; титулы: 1 «Новичок», 3 «Печатник», 5 «Переменных мастер», 8 «Укротитель циклов», 12 «Строковых дел мастер», 16 «Функциональный», 20 «Питонист» |
| «Иду ли я каждый день?» | Серия (streak) + дневная цель | цель в XP: лёгкая 15 / норм 30 / жара 60 (выбор в онбординге); заморозка серии — награда за 7 дней подряд; полоска недели ПВСЧПСВ с точками |
| «Насколько хорошо я решаю?» | Комбо точности | 3 экрана подряд с первой попытки → «Серия точных ×3» и +50% XP на следующий экран; сбивается без драмы — просто исчезает |
| «Что я умею?» | Навыки-шкалы | каждый модуль = навык («Условия», «Циклы»…); шкала растёт от экранов модуля; профиль показывает паутинку/бары навыков |
| «Что меня ждёт?» | Печати модулей + закрытые бейджи | завершение модуля = анимированная «печать главы» в коллекцию; неполученные бейджи видны силуэтами с подписью-загадкой (curiosity gap) |
| «Зачем сегодня?» | Микро-квесты дня | 1–3 в день, генерируются из типов контента: «Найди 2 бага», «Квиз без ошибок», «Урок без подсказок»; награда 15–25 XP + фрагмент заморозки (3 фрагмента = заморозка) |
| «Зачем на неделе?» | Босс недели | повторный заход в пройденного босса с усложнением (без подсказок / новые тесты) → особый бейдж и золотая рамка печати |
| «А вдруг сюрприз?» | Редкие находки | с вероятностью ~7% после экрана Пай приносит «находку»: факт о Python + 5 XP; переменная награда — деликатно, редко |
| «Что я открою?» | Разблокируемое | **темы редактора Monaco** (идеальный фит для курса кода): Solarized на 5 уровне, Dracula за первый perfect-модуль, Nord за 14 дней серии; скины Пая — за печати |

«Идеальный урок» (все экраны с первой попытки, без подсказок) — золотое кольцо
на узле урока + 10 XP бонус. Уже поддержано контентом («идеальные уроки»).

### Как не скатиться в дешёвую игру

- Нет монет, магазина и валют — только узнаваемые программистские награды
  (темы редактора!) и символы пути (печати).
- Празднования по иерархии: микро (галочка-пружинка) → средний (тост + count-up)
  → крупный (полноэкранный, 3 раза за курс на модуль). Конфетти — строго 3 повода.
- Тон взрослый: «Серия точных ×3», а не «СУПЕР-МЕГА-КОМБО!!!»

### Три варианта системы мотивации (рекомендация — гибрид 1+3)

1. **Путь** (journey): карта, главы, печати — каркас смысла.
2. **Навык** (skill): шкалы умений, паспорт Python — для «зачем».
3. **Ритм** (habit): серия, цель дня, квесты — для «каждый день».

Гибрид: путь = скелет, ритм = двигатель, навык = зеркало (в профиле).

## D. Главные экраны

### D1. Dashboard (главная)

Три варианта структуры (рекомендация — №1 с узлами из №2):

1. **«Сегодня» ★** — сверху карточка дневного фокуса, ниже путь курса.
2. **«Карта-приключение»** — полноэкранная тропа с узлами, минимум текста.
3. **«Кокпит»** — три колонки статов/пути/квестов (риск дашборда — отклонено).

Структура (сверху вниз):

```
[AppHeader: лого · полоска недели (7 точек) · серия · XP]

[Daily Focus — главная карточка]
  «Привет! Сегодня — 3 минуты»            ⌛ кольцо цели дня 18/30 XP
  Урок 2.4 · Целое деление, остаток       🔥 серия 4 дня
  ▸ шаг 2 из 3 — ты уже начал             ⚡ до уровня 5 — 12 XP
  [Продолжить обучение]                   квест дня: «Найди 2 бага» 0/2

[Путь курса — вертикальная тропа]
  Модуль-узлы соединены линией; внутри активного модуля — развёрнутые
  узлы уроков (✓ / ▶ текущий с glow / ○ будущий / 🏆 босс-узел крупнее).
  Пройденные модули свёрнуты в строку с печатью. Будущие — приглушены,
  с подписью «откроется после Главы 2» (визуально; переходы не блокируем).

[Полоска-тизер] «Глава 3 · Условия: научишь программу принимать решения»
```

Отвечает на все пять вопросов за один взгляд: где я (узел с glow), что дальше
(одна кнопка), сколько прошёл (печати + бары), что получу (до уровня 12 XP,
квест), зачем завтра (серия + тизер главы).

### D2. Экран модуля (новый, route `/module/[id]`)

Глава приключения:

```
[Хедер главы]  глиф модуля крупно (например «?»)   Глава 3 · Условия
               «Научишь программу принимать решения»
               Чему научишься: if/else · elif · логика and/or · in
               ▓▓▓▓▓░░░ 6/10 · награда главы: печать «Условия» + тема Nord

[Тропа уроков — вертикальный таймлайн]
  ✓ 3.1 if: код не для всех            идеально ◉
  ✓ 3.2 Сравнения
  ▶ 3.3 else: иначе                    [Продолжить · ≈4 мин]
  ○ 3.4 elif: много веток
  …
  🏆 БОСС: Текстовый квест — крупный узел с виньеткой и подписью
     «Собери всё в настоящую игру». До босса — 4 урока.
```

Сбоку/снизу: мини-карточка навыка («Условия — 60%»), бейджи модуля.

### D3. Экран урока (доработка текущего)

Текущая структура (хлебные крошки → заголовок → обещание → степпер →
экран → sticky-навигация) остаётся. Усиления:

- **Чекпойнт-ощущение:** степпер анимирует заполнение сегмента при завершении.
- **Блок «Частая ошибка»:** в контенте уже есть «Ловушка/Важно» — стилизуем
  blockquote-ы и абзацы с «Ловушка» в карточку с ⚠ янтарной кромкой.
- **Idle-помощь:** 60 секунд без действий в практике → Пай мягко: «Застрял?
  Первая подсказка бесплатная». Один раз за экран.
- **Завершение урока — момент награды:** полноэкранная карточка: count-up XP,
  тик серии, чек цели дня, метка «идеально» если заслужил, и тизер:
  «Следующий: elif — ещё ≈4 минуты. Продолжим?» [Дальше] [На сегодня хватит]
  Вторая кнопка — уважение к границам = доверие = возврат завтра.
- Mobile: одна колонка — задание → практика → вывод; sticky-низ остаётся;
  редактор получает кнопку «развернуть».

### D4. Onboarding (90 секунд до победы)

```
1. Обещание (5 сек):  «Научим писать на Python. Без страха. По 5 минут в день.»
   [Попробовать прямо сейчас] — без регистрации.
2. Цель (10 сек): для себя · для работы · стать разработчиком · для учёбы
   → персонализирует микрокопию («для работы» → примеры про автоматизацию).
3. Ритм (10 сек): 5 / 15 / 30 минут в день → ставит дневную цель 15/30/60 XP.
   Подпись: «Сегодня достаточно даже 5 минут».
4. Первая программа (40 сек): мини-плейграунд, код готов на 90%:
   print("Привет, ___!") — «Впиши своё имя и нажми Запустить».
   → конфетти (повод №1 из трёх) + «Ты только что написал свою первую
   программу. Серьёзно — это уже код.» + бейдж «Hello, World!»
5. Карта (10 сек): путь отрисовывается анимацией, узел 1.1 пульсирует.
   Полоска онбординга была видна все шаги: старт с 1/5, не с 0/5.
```

### D5. Профиль / прогресс

- Уровень + титул + кольцо до следующего уровня.
- Навыки: горизонтальные бары по 10 темам (от модулей).
- Печати глав: полка 10 слотов (пустые — контуром).
- Календарь-теплокарта занятий (как GitHub contributions — близко аудитории).
- Рекорды: лучшая серия, perfect-уроки, любимое время занятий.

### D6. Достижения / испытания

Две вкладки:
- **Достижения:** грид бейджей; у каждого прогресс («Багхантер 23/50»);
  закрытые — силуэт + подпись-крючок («Что-то про полночь…»).
- **Испытания:** квесты дня (1–3 карточки с прогрессом) + босс недели
  (крупная карточка с таймером до воскресенья).

## E. Визуальная система

### Цвета (токены)

| Токен | Тёмная (база) | Роль |
|---|---|---|
| `--bg` | #0B0D12 + 2 радиальных тинта (индиго 7%, зелёный 5%) | фон |
| `--surface` | #11141C | карточки |
| `--surface-2` | #181D28 | вложенные элементы, шапки карточек |
| `--line` | rgba(255,255,255,.07) / .13 hover | бордеры |
| `--text` / `--muted` | #E7EAF2 / #99A2B4 | текст |
| `--accent` | #4CC38A (REPL-зелёный) | действие, прогресс, успех |
| `--indigo` | #7A84FA | информация, ссылки активного пути |
| `--gold` | #D9B44A | XP, награды, perfect |
| `--violet` | #A78BFA | боссы, испытания |
| `--amber` | #D99A4A | предупреждения, «частая ошибка» |
| `--red` | #E5736F | только текст ошибок в консоли |
| код-поверхность | #07090D–#0E1117 | всегда тёмная, в любой теме |

Статусы: done = accent, current = accent + glow 18px/8%, locked = 40% opacity,
boss = violet, perfect = gold-кольцо.

### Типографика

- UI: Inter (или Manrope для display) — заголовки 700–750, letter-spacing −2%.
- Код: JetBrains Mono 13–14px (вкл. Monaco).
- Шкала: 12 · 13 · 15 · 17 · 20 · 24 · 30 · 40; line-height 1.55 текст / 1.15 заголовки.
- Числа XP/статов — tabular-nums.

### Сетка и форма

- Spacing база 4: 4·8·12·16·24·32·48·64. Контейнер 1120px.
- Радиусы: 8 чипы · 12 кнопки/поля · 16 карты · 999 пилюли.
- Тени почти нет; глубина — бордером и фоном. Glow только: CTA, текущий узел,
  золото perfect.

### Иконография — глифы кода (фирменный приём)

Иконки модулей — символы из самого Python, моноширинные, в плитке 46px:

`>` первые шаги · `=` переменные · `?` условия · `↻` циклы · `"a"` строки ·
`[ ]` списки · `{ }` словари · `ƒ( )` функции · `!` ошибки · `▤` файлы

UI-иконки — line-set 1.5px (Lucide). Эмодзи остаются только живым существам
и наградам: Пай 🐍, пламя 🔥, кубок 🏆. Всё остальное — глифы и иконки.

### Компоненты

- **Кнопки:** primary (accent-заливка, тёмный текст, glow), secondary
  (surface-2 + line), ghost; focus-ring 2px accent/60 + offset (a11y).
- **Карточки:** surface + line; hover: line-strong + translateY(−1px).
- **Прогресс:** бар 5–8px, скруглённый, градиент accent→#6EE7B7; кольца для
  целей (SVG, 3px). Анимация заполнения 400ms ease-out.
- **Код-блоки:** пример (шапка «ПРИМЕР · PYTHON»), результат (зелёная кромка,
  тинт 4%), плейграунд (терминальная шапка: три точки, имя файла, чип режима).
- **Бейджи:** круглые 56px, металлические оттенки по редкости (бронза/серебро/
  золото — насыщенностью, не блёстками); locked — контур.
- **Формы:** тёмные поля surface-2, focus accent; ошибок валидации — янтарь.

### Моушн

150–250ms, ease-out. Правила: движение только подтверждает действие; count-up
для XP (600ms); пламя серии «вспыхивает» scale 1→1.15→1; сегмент степпера
заливается слева направо; конфетти CSS — 3 повода; `prefers-reduced-motion` —
всё статично.

## F. Retention-механики

### Дневная петля (ядро)

```
Открыл → Daily Focus («3 минуты, шаг 2 из 3») → урок → награда (XP-count-up,
тик серии, чек цели) → тизер завтра («elif за 4 минуты») → закрыл с чувством
завершённости И одним незакрытым гештальтом (модуль 6/10).
```

### Серия и цель

- Цель дня в XP (15/30/60), видна кольцом в шапке и в Daily Focus.
- Серия растёт от любого выполненного экрана (низкий порог — защита привычки).
- Заморозка: за 7 дней подряд; максимум 2 в запасе; тратится автоматически.
- «Серия в опасности»: после 20:00 без активности — мягкий бар в шапке:
  «5 минут — и серия живёт» (когда появятся пуши — то же сообщение).

### Comeback flow (вернувшимся после 3+ дней)

- Никакого стыда. Экран: «С возвращением! Твой рекорд — 12 дней. Начнём новую
  серию с лёгкого?» → 1-экранный разогрев (простой квиз по последней теме) →
  мгновенный успех + XP → обычный урок по желанию.
- Сломанную серию не показываем красным/перечёркнутым — показываем «Рекорд: 12».

### Weekly recap (понедельник, карточка на главной)

«Твоя неделя: 240 XP · 5 дней · 12 уроков · сильнее всего вырос навык „Циклы“ ·
на 30% больше практики, чем неделей раньше». Кнопка «Поделиться» (картинка).

### Reward loops

- Быстрый: экран → XP-тост (1.8с) → сегмент степпера заполняется.
- Сессионный: урок → celebration + чек цели дня.
- Недельный: квесты → заморозка; босс недели → бейдж.
- Долгий: модуль → печать + разблокировка темы редактора; курс → сертификат.

## J. Финальный результат

### 1. Концепция одной строкой
«Терминал, который стал тёплым»: путь из 10 глав по тёмной тропе кода, где
каждый день — маленькая победа за 5 минут, каждая глава — печать в коллекцию,
а наставник Пай превращает ошибки из страха в диалог.

### 2. Улучшения по приоритету

**P0 — ядро привычки (1-я итерация)**
1. Онбординг 90 секунд с первой программой и конфетти.
2. Lesson-complete celebration (count-up XP, серия, цель, тизер).
3. Цель дня + кольцо в шапке + точки недели.
4. Lock/current/perfect-состояния на тропе; Daily Focus вместо простого hero.
5. Комбо точности (×XP) — мгновенная дофаминовая петля.

**P1 — смысл и возврат (2-я итерация)**
6. Страница модуля-«главы» + печати модулей.
7. Квесты дня + заморозки серии.
8. Профиль: навыки, теплокарта, рекорды.
9. Бейджи на поверхности (грид с прогрессом, силуэты закрытых).
10. Weekly recap + comeback flow.

**P2 — wow и масштаб (3-я итерация)**
11. Разблокируемые темы Monaco + скины Пая.
12. Босс недели; редкие находки.
13. Светлая тема (код остаётся тёмным); лиги — только если появятся аккаунты.

### 3. Пошаговый план внедрения
Шаг 1: токены/глифы/типографика (полдня) → Шаг 2: ProgressProvider + расширение
стора (день) → Шаг 3: P0-фичи (3–4 дня) → Шаг 4: страница модуля + печати
(2 дня) → Шаг 5: квесты/профиль/бейджи (3 дня) → Шаг 6: recap, comeback, темы
(2–3 дня). Каждый шаг — вертикальный срез, продукт живой всегда.

### 4. Must-have
Онбординг-победа за 90 сек · одна CTA на экран · цель дня + серия в шапке ·
celebration после урока · «≈ N мин» у каждого действия · дружелюбные ошибки ·
сегментный степпер · тёмные код-поверхности всегда · count-up XP · comeback без стыда.

### 5. Wow-features
Глифы-иконки из кода · печати глав с анимацией · разблокируемые темы редактора ·
комбо точности · теплокарта занятий · босс недели · редкие находки Пая ·
режим «Играть» у боссов (уже есть — поверхность для маркетинга).

### Лучшие UX-паттерны edtech (применённые)
Chunking 3–5 мин (есть) · мгновенный фидбек (есть) · нормализация ошибок ·
прогрессивные подсказки с честной ценой · goal gradient · endowed progress ·
variable rewards (редко) · implementation intentions («3 мин») · spaced
repetition через квесты-разминки из старых модулей · Zeigarnik в Daily Focus.
