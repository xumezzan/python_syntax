### Урок 6.1 — Что такое список
**Экраны:** теория → допиши → квиз

#### 📖 Теория
Список — это коробка с пронумерованными отделениями. Хранит сколько угодно значений:

```python
games = ["шахматы", "доту", "майнкрафт"]
print(games[0])     # шахматы
print(games[-1])    # майнкрафт
print(games)        # ['шахматы', 'доту', 'майнкрафт']
```

Квадратные скобки, элементы через запятую. Индексы работают ровно как у строк: с нуля, минус — с конца. В одном списке могут жить разные типы: `[42, "привет", True]` — законно (но обычно не нужно).

#### ✏️ Допиши код
**Условие:** выведи второй по счёту предмет рюкзака.
**Стартовый код:**
```python
backpack = ["ноутбук", "бутылка", "зонт"]
print(backpack[___])
```
**Подсказки:**
1. Второй по счёту — это какой индекс при отсчёте с нуля?
2. Индекс 1.
3. Решение: `print(backpack[1])` → `бутылка`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `бутылка` |

#### ❓ Квиз
Что выведет код?
```python
nums = [10, 20, 30, 40]
print(nums[1] + nums[-1])
```
- `50` — ❌ nums[1] — это 20, не 10.
- ✅ `60`
- `30` — ❌ nums[-1] — последний элемент, 40.
- `[20, 40]` — ❌ плюс между числами — сложение, не объединение.

### Урок 6.2 — Изменение, len, in
**Экраны:** теория → допиши → задача

#### 📖 Теория
Главное отличие списка от строки: список можно менять прямо на месте:

```python
team = ["Аня", "Борис", "Вика"]
team[1] = "Богдан"          # замена по индексу
print(team)                  # ['Аня', 'Богдан', 'Вика']
print(len(team))             # 3
print("Аня" in team)         # True
```

Старые знакомые работают и тут: `len()` — сколько элементов, `in` — есть ли такой элемент.

playlist = ["старая попса", "очень старая попса", "классика"]

print(playlist)

**Подсказки:**
1. Присваивание по индексу: имя[номер] = значение.
2. `playlist[1]`.
3. Решение: `playlist[1] = "новый хит"`.
**Автотесты:**
| Ввод | Ожидаемый вывод |

### Урок 6.3 — append, insert, remove, pop
**Экраны:** теория → квиз → задача

#### 📖 Теория
Методы изменения списка:

```python
todo = ["поесть"]
todo.append("поспать")       # добавить в конец → ['поесть', 'поспать']
todo.insert(0, "проснуться") # вставить по индексу → ['проснуться', 'поесть', 'поспать']
todo.remove("поесть")        # удалить по значению → ['проснуться', 'поспать']
last = todo.pop()            # достать последний → last = 'поспать'
```

`append` — твой главный рабочий инструмент: 95% добавлений — в конец. Заметь: эти методы меняют сам список — ничего присваивать не нужно (`todo = todo.append(...)` — ошибка, так список потеряется!).

#### ❓ Квиз
Что выведет код?
```python
nums = [1, 2, 3]
nums.append(4)
nums.remove(2)
print(nums)
```
- `[1, 2, 3, 4]` — ❌ remove(2) удалил двойку.
- ✅ `[1, 3, 4]`
- `[1, 4]` — ❌ remove удаляет ЗНАЧЕНИЕ 2, а не два элемента.
- `[2, 3, 4]` — ❌ remove ищет по значению, не по индексу.

#### 💻 Задача
**Условие:** очередь в кофейне. Начни с пустого списка. Читай имена, пока не встретишь `открылись`. Каждое имя добавляй в очередь. В конце выведи список и первого счастливчика. Формат вывода — две строки: список и `Первый: имя`.
**Подсказки:**
1. `queue = []`, цикл while True с break, потом два print.
2. Пустой список — это `[]`. Первый — `queue[0]`.
3. Решение:
```python
queue = []
while True:
    name = input()
    if name == "открылись":
        break
    queue.append(name)
print(queue)
print(f"Первый: {queue[0]}")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `Ян` ⏎ `Ия` ⏎ `открылись` | `['Ян', 'Ия']` ⏎ `Первый: Ян` |

### Урок 6.4 — Цикл по списку
**Экраны:** теория → допиши → задача

#### 📖 Теория
`for` ходит по списку так же, как по строке:

```python
prices = [100, 250, 80]
for price in prices:
    print(f"Товар за {price} руб.")
```

Имя переменной выбирай говорящее: `for price in prices` читается как книга. Все паттерны из модуля 4 — накопитель, счётчик, поиск максимума — работают со списками без изменений.

#### ✏️ Допиши код
**Условие:** поприветствуй каждого ученика класса.
**Стартовый код:**
```python
pupils = ["Ким", "Зоя", "Лев"]
for ___ in ___:
    print(f"Привет, {pupil}!")
```
**Подсказки:**
1. Схема: for <переменная> in <список>.
2. Переменная должна называться `pupil` — она уже используется в print.
3. Решение: `for pupil in pupils:`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Привет, Ким!` ⏎ `Привет, Зоя!` ⏎ `Привет, Лев!` |

#### 💻 Задача
**Условие:** оценки за четверть: `[5, 4, 3, 5, 4, 5]` (задай в коде). Посчитай и выведи, сколько пятёрок.
**Подсказки:**
1. Счётчик + цикл + if.
2. `if grade == 5: count += 1`.
3. Решение:
```python
grades = [5, 4, 3, 5, 4, 5]
count = 0
for grade in grades:
    if grade == 5:
        count += 1
print(count)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `3` |

### Урок 6.5 — sum, min, max, sort
**Экраны:** теория → квиз → задача

#### 📖 Теория
Для частых задач у Python есть готовые функции — не нужно писать циклы:

```python
steps = [8200, 12500, 4300, 9800]
print(sum(steps))      # 34800 — сумма
print(max(steps))      # 12500 — максимум
print(min(steps))      # 4300  — минимум
print(len(steps))      # 4

steps.sort()           # сортировка НА МЕСТЕ (меняет список)
print(steps)           # [4300, 8200, 9800, 12500]
steps.sort(reverse=True)  # по убыванию
```

Средний шаг за день? `sum(steps) / len(steps)`. Красота.

#### ❓ Квиз
Что выведет код?
```python
nums = [3, 1, 2]
nums.sort()
print(nums[0])
```
- `3` — ❌ после sort список стал [1, 2, 3].
- ✅ `1`
- `2` — ❌ nums[0] — первый элемент отсортированного списка, минимальный.
- `[1, 2, 3]` — ❌ печатается один элемент, не весь список.

#### 💻 Задача
**Условие:** прочитай n, затем n чисел — цены одного товара в разных магазинах. Выведи две строки: `Дешевле всего: X` и `Разница: Y`, где Y — разница между максимальной и минимальной ценой.
**Подсказки:**
1. Собери цены в список через append, потом min/max.
2. Числа читай как `int(input())` в цикле for.
3. Решение:
```python
n = int(input())
prices = []
for i in range(n):
    prices.append(int(input()))
print(f"Дешевле всего: {min(prices)}")
print(f"Разница: {max(prices) - min(prices)}")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `3` ⏎ `500` ⏎ `450` ⏎ `520` | `Дешевле всего: 450` ⏎ `Разница: 70` |

### Урок 6.6 — split и join: строка ↔ список
**Экраны:** теория → допиши → задача

#### 📖 Теория
Два метода-моста между строками и списками:

sentence = "я учу питон"

data = "5,8,13".split(",")      # ['5', '8', '13'] — по любому разделителю

glued = "-".join(["а", "б", "в"])  # "а-б-в" — склеить через разделитель

`split` особенно важен для ввода: «числа в одну строку через пробел» читаются так:

nums = input().split()    # внимание: это список СТРОК — ['5', '8']

### Урок 6.7 — Срезы списков и копии
**Экраны:** теория → квиз → баг

#### 📖 Теория
Срезы работают и со списками:

```python
top10 = ["а", "б", "в", "г", "д", "е", "ж", "з", "и", "к"]
print(top10[:3])     # ['а', 'б', 'в'] — топ-3
print(top10[-2:])    # ['и', 'к'] — два последних
```

А теперь ловушка века:

```python
a = [1, 2, 3]
b = a            # это НЕ копия! Это вторая наклейка на ТУ ЖЕ коробку
b.append(4)
print(a)         # [1, 2, 3, 4] — а изменился тоже!
```

Хочешь настоящую копию — возьми полный срез: `b = a[:]` (или `a.copy()`).

#### ❓ Квиз
Что выведет код?
```python
x = [1, 2]
y = x
y[0] = 99
print(x)
```
- `[1, 2]` — ❌ y — не копия, а то же самый список под вторым именем.
- ✅ `[99, 2]`
- `[99, 99]` — ❌ менялся только нулевой элемент.
- `Ошибка` — ❌ всё законно, в этом и коварство.

#### 🐞 Найди баг
**Код:**
```python
original = ["король", "ферзь"]
backup = original
original.append("пешка")
print(f"Резервная копия: {backup}")
```
**Задумка:** backup должен был остаться `['король', 'ферзь']`.
**Ошибка:** строка 2 — `backup = original` не копирует.
**Правильно:** `backup = original[:]` или `backup = original.copy()`.

### Урок 6.8 — List comprehension
**Экраны:** теория → допиши → задача

#### 📖 Теория
Очень частая задача: пройди по списку, преобразуй каждый элемент, собери новый список. Длинный путь ты знаешь. Короткий — **list comprehension**:

```python
nums = [1, 2, 3, 4]
squares = [n * n for n in nums]          # [1, 4, 9, 16]
big = [n for n in nums if n > 2]         # [3, 4] — ещё и фильтр!
```

Читай изнутри: «возьми `n * n` для каждого `n` из `nums`». Это та же самая запись for, свёрнутая в одну строку внутри скобок.

Классика с вводом — превратить строки в числа:

```python
nums = [int(x) for x in input().split()]
```

#### ✏️ Допиши код
**Условие:** из списка имён сделай список их длин.
**Стартовый код:**
```python
names = ["Ян", "Мария", "Пётр"]
lengths = [___ for name in names]
print(lengths)
```
**Подсказки:**
1. Что нужно вычислить для каждого имени? Его длину.
2. `len(name)`.
3. Решение: `lengths = [len(name) for name in names]` → `[2, 5, 4]`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `[2, 5, 4]` |

#### 💻 Задача
**Условие:** прочитай числа в одну строку через пробел. Выведи список только чётных из них (как список). Ввод `1 4 7 8 2` → `[4, 8, 2]`. Реши одной comprehension (плюс input и print).
**Подсказки:**
1. Сначала преврати ввод в числа, потом отфильтруй. Можно в две comprehension или в одну.
2. `[n for n in nums if n % 2 == 0]`.
3. Решение:
```python
nums = [int(x) for x in input().split()]
print([n for n in nums if n % 2 == 0])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `1 4 7 8 2` | `[4, 8, 2]` |
| `1 3 5` | `[]` |

🏅 За решение через comprehension — бейдж **«Однострочник»**.

### Урок 6.9 — Кортежи
**Экраны:** теория → квиз → допиши

#### 📖 Теория
Кортеж (`tuple`) — это список, который нельзя менять. Круглые скобки вместо квадратных:

```python
point = (10, 20)
print(point[0])      # 10 — чтение как у списка
point[0] = 5         # TypeError! менять нельзя
```

Зачем нужен «список-инвалид»? Для данных, которые не должны меняться: координаты, даты, RGB-цвета. Python и сам их любит — помнишь `range`?

Самое вкусное — **распаковка**:

```python
x, y = point         # x = 10, y = 20 — одним махом!
a, b = b, a          # обмен значений без временной переменной. Магия!
```

#### ❓ Квиз
Что выведет код?
```python
a = 1
b = 2
a, b = b, a
print(a, b)
```
- `1 2` — ❌ распаковка обменяла значения.
- ✅ `2 1`
- `2 2` — ❌ Python вычисляет правую часть ДО присваивания — ничего не затирается.
- `Ошибка` — ❌ это легальный и любимый питонистами трюк.

#### ✏️ Допиши код
**Условие:** распакуй кортеж с координатами клада.
**Стартовый код:**
```python
treasure = (35, 78)
___, ___ = treasure
print(f"Шаг на восток: {x}, шаг на север: {y}")
```
**Подсказки:**
1. Слева от = — две переменные через запятую.
2. Имена `x` и `y` — они уже в print.
3. Решение: `x, y = treasure`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Шаг на восток: 35, шаг на север: 78` |

### Урок 6.10 — Марафон повторения
**Экраны:** квиз → баг → задача

#### ❓ Квиз
Что выведет код?
```python
words = "до ре ми".split()
words.append("фа")
print(len(words))
```
- `3` — ❌ после split было 3, но append добавил элемент.
- ✅ `4`
- `8` — ❌ len списка считает элементы, а не буквы.
- `Ошибка` — ❌ split вернул список, append к нему применим.

#### 🐞 Найди баг
**Код:**
```python
cart = ["хлеб", "сыр"]
cart = cart.append("чай")
print(cart)
```
**Ошибка:** строка 2 — `append` возвращает `None`, присваивание затёрло список.
**Вывод:** `None`
**Правильно:** просто `cart.append("чай")` без присваивания.
**Объяснение:** методы, меняющие список на месте (append, sort, remove), ничего не возвращают. Присваивать их результат — значит потерять список.

#### 💻 Задача
**Условие:** прочитай результаты забега — времена в секундах в одну строку через пробел. Выведи тройку призёров (три наименьших времени) списком. Ввод `12 10 15 9 14` → `[9, 10, 12]`.
**Подсказки:**
1. Преврати в числа, отсортируй, возьми срез.
2. `nums.sort()`, потом `nums[:3]`.
3. Решение:
```python
nums = [int(x) for x in input().split()]
nums.sort()
print(nums[:3])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `12 10 15 9 14` | `[9, 10, 12]` |
| `3 2 1` | `[1, 2, 3]` |

### Урок 6.11 — 🏆 БОСС: Топ-чарт фильмов
**Экраны:** босс (50 XP)

#### 📖 Вступление
🐍 Пай: «Построим свой Кинопоиск! Программа-чарт: добавляем фильмы с оценками, смотрим топ».

#### 💻 Задание
**Условие:** программа в цикле читает команды:
- `добавить <оценка> <название из одного слова>` — например `добавить 9 Интерстеллар` — добавляет фильм;
- `топ` — выводит фильмы по убыванию оценки в формате `1. Название (9)` (при равных оценках порядок любой стабильный);
- `выход` — печатает `Пока!` и завершается.

**Пример сессии:**
```
Ввод:  добавить 9 Интерстеллар
Ввод:  добавить 7 Барби
Ввод:  добавить 10 Шрек
Ввод:  топ
Вывод: 1. Шрек (10)
       2. Интерстеллар (9)
       3. Барби (7)
Ввод:  выход
Вывод: Пока!
```

**Подсказки:**
1. Храни фильмы списком кортежей `(оценка, название)` — такие кортежи сортируются прямо по оценке. Команду разбирай через `split()`.
2. Каркас: `parts = line.split()`; если `parts[0] == "добавить"` → `films.append((int(parts[1]), parts[2]))`; команда `топ` → `films.sort(reverse=True)` и цикл с нумерацией.
3. Решение:
```python
films = []
while True:
    parts = input().split()
    if parts[0] == "выход":
        print("Пока!")
        break
    elif parts[0] == "добавить":
        films.append((int(parts[1]), parts[2]))
    elif parts[0] == "топ":
        films.sort(reverse=True)
        n = 1
        for score, title in films:
            print(f"{n}. {title} ({score})")
            n += 1
```
