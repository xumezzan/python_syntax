# 🐍 PySyntax — платформа обучения Python: от синтаксиса до enterprise-интеграций

Интерактивные треки: теория + код в браузере + автотесты + подсказки + XP/стрики.

**Два трека:**
1. **Python-синтаксис с нуля** — 10 модулей, 99 уроков (главная страница).
2. **Python Backend: API, базы данных и интеграции** — 18 модулей, 128 уроков
   (`/track/backend`). Завершается реальным проектом `enterprise-integration/`.

## Структура

```
python_syntax/
├── docs/DESIGN.md          ← дизайн-система и продуктовая стратегия
├── enterprise-integration/ ← финальный проект backend-трека: FastAPI + PostgreSQL
│                             + SQLAlchemy + Alembic + JWT + Docker + pytest (mock SAP/банк)
└── platform/               ← веб-приложение (Next.js + Monaco + Pyodide)
    ├── content/            ← контент синтаксис-курса (10 модулей, markdown)
    │   ├── 00-структура-курса.md   — формат уроков, правила XP, бейджи
    │   ├── module-01..10.md        — уроки синтаксис-курса
    │   └── backend/                ← контент backend-трека (18 модулей)
    │       ├── 00-структура.md     — карта трека и особенности авто-проверки
    │       └── module-01..18.md    — HTTP, JSON, REST, FastAPI, SQL, ORM, …, SAP, финал
    ├── app/ components/ lib/       — код платформы (треки, карта, плеер, профиль)
    └── public/py-worker.js         — Python-песочница в Web Worker
```

## Учебные треки

- **Главная (`/`)** — синтаксис-курс + карточка-баннер «Что дальше» с переходом
  в backend-трек.
- **Роудмап (`/roadmap`)** — путь обучения: синтаксис → backend → финальный проект,
  с прогрессом по каждому треку.
- **Страница трека (`/track/backend`)** — герой с роудмапом, блок «зачем в работе»,
  карточки модулей со статусами (не начат / в процессе / завершён), карточка
  финального проекта и чеклист навыков.
- Прогресс, XP, уровни, бейджи — единые на оба трека (`localStorage`). Глобальные id
  модулей сквозные (синтаксис 1–10, backend 11–28), поэтому ключи прогресса и роуты
  `/module/[id]`, `/lesson/[m]/[l]` не конфликтуют, а старый курс не ломается.


## Деплой на Railway

1. Создай проект из этого GitHub-репозитория.
2. В настройках сервиса: **Root Directory = `platform`** (Next.js определится сам,
   `railway.json` подхватится оттуда же).
3. Подключи **Volume** (например, в `/data`) и задай переменную окружения
   **`DATA_DIR=/data`** — там живёт файловое хранилище синхронизации; без
   volume коды синхронизации будут стираться при каждом деплое.
4. Порт прокидывается автоматически (`next start` слушает `$PORT`).

Pyodide и Monaco грузятся с CDN на клиенте — серверу хватает минимального
инстанса.

Нужен интернет: редактор Monaco и Python-среда Pyodide загружаются с CDN
при первом открытии урока (~10–20 сек, дальше кэшируется).

## Как это устроено

- **Контент = markdown.** `platform/lib/course.mjs` парсит файлы из `content/`
  в структуру экранов: теория / допиши код / задача / квиз / найди баг / собери код.
  Правишь markdown — урок обновляется без пересборки.
- **Python в браузере.** Pyodide (CPython → WebAssembly): код ученика выполняется
  локально, без сервера. `input()` кормится строками из тестов или поля «Ввод»;
  когда строки кончаются — EOF (страховка от вечных циклов чтения).
- **Автотесты** — таблицы «ввод → ожидаемый вывод» из markdown; сравнение
  построчное. Задачи без машинных тестов работают в режиме самопроверки.
- **Геймификация** — XP, уровни с титулами, стрик, цель дня, комбо точности
  (×1.5 XP с 3-го верного ответа подряд), идеальные уроки, бейджи; всё в
  localStorage (`pysyntax-v1`). Новичка встречает онбординг с первой
  программой за 90 секунд (`/start`).
- **Дружелюбные ошибки** — типовые traceback переводятся на русский
  (`platform/lib/pyodide.js`, функция `friendlyError`).

### Backend-трек в браузере

Pyodide включает всю стандартную библиотеку (`json`, `sqlite3`, `xml.etree`,
`base64`, `hashlib`), но не `fastapi`/`sqlalchemy`/сеть. Поэтому авто-тестируемые
задачи backend-трека написаны на чистом Python (разбор JSON/XML, SQL на sqlite,
retry/idempotency, маппинг) — они реально запускаются. Темы FastAPI/SQLAlchemy/
Docker/Git идут через квизы, «найди баг», «собери код» и самопроверку, а живой
стек — в проекте `enterprise-integration/`.

## Проверка контента

```bash
cd platform
npm run check                       # сводка по трекам + проверка минимумов
node scripts/verify-solutions.mjs   # прогон эталонных решений backend-задач (нужен python3)
```

`npm run check` выводит по каждому треку число уроков, экранов и тест-кейсов и
проверяет минимумы (≥3 урока, ≥2 практики, ≥3 квиза на модуль).
`verify-solutions.mjs` исполняет эталонные решения авто-задач и сверяет вывод.

## Финальный проект — Enterprise Integration Simulator

`enterprise-integration/` — рабочий backend (FastAPI + PostgreSQL + SQLAlchemy +
Alembic + JWT + Docker + pytest), связывающий **mock-SAP** и **mock-банк**:
создание платежей, отправка в банк, webhook, integration logs, idempotency, retry.
Настоящих SAP и банка нет — только безопасные моки, секреты только через `.env`.

```bash
cd enterprise-integration
cp .env.example .env
docker compose up --build           # app + PostgreSQL; документация на /docs
# или локально без Docker (sqlite по умолчанию):
pip install -r requirements.txt && uvicorn app.main:app --reload
pytest                              # 22 теста
alembic upgrade head                # миграции (при AUTO_CREATE_TABLES=false)
```

Полное описание, endpoints и сценарий «жизнь платежа» — в
`enterprise-integration/README.md`.

## Что дальше (за рамками прототипа)

Полная дизайн-стратегия — в `docs/DESIGN.md`. Реализованы приоритеты P0 и P1:
онбординг, celebration, цель дня, комбо, Daily Focus и состояния тропы;
страницы главы (`/module/[id]`) и профиля (`/profile`) с навыками, печатями,
теплокартой и достижениями; квесты дня с фрагментами и заморозками серии
(спасают серию при пропуске одного дня); comeback-карточка и weekly recap.

Из P2 реализовано: разблокируемые темы Monaco (Monokai/Nord/Solarized/Dracula —
открываются за босса, серию, уровень и идеальные уроки; выбор в профиле),
босс недели (+40 XP и бейдж «Вызов принят»), списание XP за подсказки
(−5/−15), выполнение кода в Web Worker с таймаутом 10 секунд — вечный цикл
убивает воркер, а не вкладку (интерактивный режим «Играть» остаётся в главном
потоке ради диалогов).

Также реализованы: светлая тема с переключателем в шапке (фирменный паттерн
«код живёт в ночи» — все кодовые поверхности остаются тёмными в любой теме)
и «Разминка» (`/warmup`) — spaced repetition: 3 случайных квиза из пройденных
уроков, +5 XP за каждый, не больше раза в день; туда же ведёт comeback-карточка.

Финальный срез: **синхронизация устройств** без регистрации — анонимный код
(`pys-xxxxxxxx`) в профиле, прогресс автоматически пушится на сервер
(`/api/sync`, файловое хранилище `platform/data/`) и восстанавливается по коду
на другом устройстве; уровень безопасности прототипа — код = доступ, как
ссылка-шеринг. И **скины Пая**: 🐍 → 🐢 (10 уроков) → 🐲 (3 босса) → 👑
(5 идеальных) — применяются в логотипе, коллаутах теории и сообщениях об ошибках.

Дальше (потребует продакшен-инфраструктуры):

- настоящие аккаунты (OAuth) и БД вместо файлового хранилища;
- пуш-напоминания о серии (service worker + VAPID).
