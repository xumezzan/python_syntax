### Урок 3.1 — if: код не для всех
**Экраны:** теория → допиши → задача

#### 📖 Теория
До сих пор выполнялась каждая строка. Команда `if` делает код условным — он сработает, только если условие истинно:

```python
temperature = 35
if temperature > 30:
    print("Жара! Бери воду")
print("Хорошего дня")
```
```
Жара! Бери воду
Хорошего дня
```

Три правила:
1. После условия — двоеточие `:`
2. Условный код сдвинут вправо на **4 пробела** — это называется отступ
3. Всё, что с отступом, — внутри if; первая строка без отступа — уже снаружи

Отступы — не украшение, а грамматика Python. Так он понимает, где условный блок заканчивается.

#### ✏️ Допиши код
**Условие:** если на счету больше 100 монет — программа должна хвастаться. Допиши условие.
**Стартовый код:**
```python
coins = 150
if coins ___ 100:
    print("Я богат!")
```
**Подсказки:**
1. «Больше» — такой же знак, как в математике.
2. Оператор `>`.
3. Решение: `if coins > 100:` — 150 > 100, поэтому строка выведется.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Я богат!` |

#### 💻 Задача
**Условие:** прочитай число — температуру за окном. Если она ниже 0, выведи `Надень шапку!`. В любом случае последней строкой выведи `Пока!`.
**Подсказки:**
1. `int(input())`, потом if с `< 0`, потом print без отступа.
2. `print("Пока!")` должен быть БЕЗ отступа — он выполняется всегда.
3. Решение:
```python
t = int(input())
if t < 0:
    print("Надень шапку!")
print("Пока!")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `-5` | `Надень шапку!` ⏎ `Пока!` |
| `10` | `Пока!` |
| `0` | `Пока!` |

### Урок 3.2 — Сравнения
**Экраны:** теория → квиз → баг

#### 📖 Теория
Шесть операторов сравнения:

| Оператор | Значение | Пример |
|----------|----------|--------|
| `==` | равно | `5 == 5` → True |
| `!=` | не равно | `5 != 3` → True |
| `>` `<` | больше / меньше | `7 > 2` → True |
| `>=` `<=` | больше/меньше или равно | `5 >= 5` → True |

**Ловушка №1 всех новичков:** `=` кладёт в переменную, `==` сравнивает. Перепутаешь — Python ругнётся:

```python
if x = 5:    # SyntaxError!
if x == 5:   # правильно
```

Результат сравнения — это значение типа `bool`: `True` или `False`. Его можно даже напечатать: `print(3 > 1)` выведет `True`.

#### ❓ Квиз
Что выведет `print(10 != 10)`?
- `True` — ❌ `!=` значит «НЕ равно», а 10 равно 10.
- ✅ `False`
- `10` — ❌ выводится результат сравнения, не число.
- `Ошибка` — ❌ сравнение корректное.

#### 🐞 Найди баг
**Код:**
```python
password = input()
if password = "1234":
    print("Доступ открыт")
```
**Ошибка:** строка 2 — одиночное `=` вместо `==`.
**Правильно:** `if password == "1234":`
**Объяснение:** запомни мантру: «одно равно — кладу, два равно — сравниваю».

### Урок 3.3 — else: иначе
**Экраны:** теория → допиши → задача

#### 📖 Теория
`else` — это «во всех остальных случаях». Сработает либо ветка if, либо ветка else — всегда ровно одна:

```python
age = 16
if age >= 18:
    print("Проходите")
else:
    print("Вход с 18 лет")
```
```
Вход с 18 лет
```

`else` пишется без условия (оно и так понятно — «всё остальное»), но с двоеточием. И его блок тоже с отступом.

#### ✏️ Допиши код
**Условие:** допиши вторую ветку: если число не делится на 2 — оно нечётное.
**Стартовый код:**
```python
n = int(input())
if n % 2 == 0:
    print("Чётное")
___:
    print("Нечётное")
```
**Подсказки:**
1. Нужно слово «иначе» по-английски.
2. `else` — и не забудь двоеточие.
3. Решение: `else:` — помнишь `%` из модуля 2? Остаток 0 — значит делится.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `8` | `Чётное` |
| `7` | `Нечётное` |
| `0` | `Чётное` |

#### 💻 Задача
**Условие:** магазин дарит скидку при покупке от 1000 рублей. Прочитай сумму. Если она `>= 1000` — выведи `Скидка 10%`, иначе — `Скидки нет`.
**Подсказки:**
1. if/else с условием `>= 1000`.
2. Внимание на «от 1000»: ровно 1000 — это уже скидка, нужен `>=`, не `>`.
3. Решение:
```python
total = int(input())
if total >= 1000:
    print("Скидка 10%")
else:
    print("Скидки нет")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `1500` | `Скидка 10%` |
| `999` | `Скидки нет` |
| `1000` | `Скидка 10%` |

### Урок 3.4 — elif: много веток
**Экраны:** теория → допиши → задача

#### 📖 Теория
Когда вариантов больше двух, между `if` и `else` встают ветки `elif` («иначе если»):

```python
score = 75
if score >= 90:
    print("Отлично")
elif score >= 70:
    print("Хорошо")
elif score >= 50:
    print("Сойдёт")
else:
    print("Попробуй ещё")
```
```
Хорошо
```

Python проверяет условия сверху вниз и выполняет **первое** подошедшее — остальные даже не смотрит. Поэтому 75 дало «Хорошо», хотя 75 тоже `>= 50`.

#### ✏️ Допиши код
**Условие:** светофор! Допиши среднюю ветку для жёлтого.
**Стартовый код:**
```python
light = input()
if light == "красный":
    print("Стой")
___ light == "жёлтый":
    print("Приготовься")
else:
    print("Иди")
```
**Подсказки:**
1. «Иначе если» — одно слово из четырёх букв.
2. `elif` — гибрид else + if.
3. Решение: `elif light == "жёлтый":`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `красный` | `Стой` |
| `жёлтый` | `Приготовься` |
| `зелёный` | `Иди` |

#### 💻 Задача
**Условие:** категории фильмов по возрасту. Прочитай возраст и выведи: до 6 — `0+`, до 12 — `6+`, до 16 — `12+`, до 18 — `16+`, иначе — `18+`. («До 6» значит `age < 6`.)
**Подсказки:**
1. Цепочка if/elif/elif/elif/else, условия `< 6`, `< 12`, `< 16`, `< 18`.
2. Порядок важен: проверяй от меньшего к большему — первое сработавшее условие побеждает.
3. Решение:
```python
age = int(input())
if age < 6:
    print("0+")
elif age < 12:
    print("6+")
elif age < 16:
    print("12+")
elif age < 18:
    print("16+")
else:
    print("18+")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `5` | `0+` |
| `12` | `12+` |
| `17` | `16+` |
| `40` | `18+` |

### Урок 3.5 — Условие внутри условия
**Экраны:** теория → квиз → задача

#### 📖 Теория
Внутри if может жить ещё один if — просто добавь ещё 4 пробела отступа:

```python
has_ticket = True
age = 20
if has_ticket:
    if age >= 18:
        print("Добро пожаловать!")
    else:
        print("Этот фильм 18+")
else:
    print("Сначала купи билет")
```

Заметь: `if has_ticket:` — без всякого сравнения! `has_ticket` уже `True` или `False`, ему не с чем сравниваться. Писать `if has_ticket == True:` можно, но это как сказать «правда ли, что правда» — лишнее.

#### ❓ Квиз
Что выведет код?
```python
x = 10
if x > 5:
    if x > 20:
        print("A")
    else:
        print("B")
else:
    print("C")
```
- `A` — ❌ 10 не больше 20, внутренний if уйдёт в else.
- ✅ `B`
- `C` — ❌ 10 > 5, поэтому внешний if сработал.
- Ничего — ❌ какая-то ветка выполнится всегда: у обоих if есть else.

#### 💻 Задача
**Условие:** банкомат. Прочитай две строки: PIN и сумму. Если PIN не равен `4321` — выведи `Неверный PIN`. Если PIN верный, но сумма больше 5000 — `Лимит 5000`. Если всё хорошо — `Выдаю деньги`.
**Подсказки:**
1. Внешний if проверяет PIN, внутренний — сумму.
2. PIN сравнивай как строку: `pin == "4321"` (вводы с нулём в начале типа "0042" ломаются на int).
3. Решение:
```python
pin = input()
amount = int(input())
if pin == "4321":
    if amount > 5000:
        print("Лимит 5000")
    else:
        print("Выдаю деньги")
else:
    print("Неверный PIN")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `4321` ⏎ `1000` | `Выдаю деньги` |
| `4321` ⏎ `9000` | `Лимит 5000` |
| `1111` ⏎ `100` | `Неверный PIN` |

### Урок 3.6 — and и or
**Экраны:** теория → допиши → задача

#### 📖 Теория
Два условия можно объединить:

- `and` — «и»: истинно, когда **оба** условия истинны
- `or` — «или»: истинно, когда **хотя бы одно** истинно

```python
age = 25
if age >= 14 and age < 60:
    print("Рабочий возраст")

day = "суббота"
if day == "суббота" or day == "воскресенье":
    print("Выходной!")
```

Часто `and` помогает проверить диапазон. Кстати, у Python есть и красивая короткая форма: `14 <= age < 60` — как в учебнике математики. Обе записи правильные.

#### ✏️ Допиши код
**Условие:** на аттракцион пускают тех, кто старше 10 лет И выше 140 см. Допиши оператор.
**Стартовый код:**
```python
age = int(input())
height = int(input())
if age > 10 ___ height > 140:
    print("Проходи!")
else:
    print("Извини, нельзя")
```
**Подсказки:**
1. Должны выполниться ОБА условия сразу.
2. Оператор «и» — `and`.
3. Решение: `if age > 10 and height > 140:`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `12` ⏎ `150` | `Проходи!` |
| `12` ⏎ `130` | `Извини, нельзя` |
| `8` ⏎ `150` | `Извини, нельзя` |

#### 💻 Задача
**Условие:** кафе закрыто в понедельник и вторник. Прочитай день недели и выведи `Закрыто` или `Открыто`.
**Подсказки:**
1. Два сравнения строки, объединённые так, чтобы хватило одного совпадения.
2. `day == "понедельник" or day == "вторник"`.
3. Решение:
```python
day = input()
if day == "понедельник" or day == "вторник":
    print("Закрыто")
else:
    print("Открыто")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `понедельник` | `Закрыто` |
| `вторник` | `Закрыто` |
| `пятница` | `Открыто` |

### Урок 3.7 — not и ловушки логики
**Экраны:** теория → квиз → баг

#### 📖 Теория
`not` переворачивает условие:

```python
is_raining = False
if not is_raining:
    print("Зонт не нужен")
```

И знаменитая ловушка с `or`. По-русски говорят «если x равен 1 или 2». Хочется написать:

```python
if x == 1 or 2:      # ВСЕГДА истинно! Ловушка!
```

Python читает это как `(x == 1) or (2)`, а непустое число `2` для него — «правда». Правильно — сравнивать каждый раз:

```python
if x == 1 or x == 2:   # вот так
```

#### ❓ Квиз
Что выведет код?
```python
x = 5
if not x > 10:
    print("Маленькое")
else:
    print("Большое")
```
- `Большое` — ❌ `x > 10` — это False, а `not False` — True.
- ✅ `Маленькое`
- `Ошибка` — ❌ `not` перед сравнением — законная конструкция.
- Ничего — ❌ одна из веток if/else выполняется всегда.

#### 🐞 Найди баг
**Код:**
```python
answer = input()
if answer == "да" or "ага":
    print("Договорились!")
else:
    print("Ну нет так нет")
```
**Ошибка:** строка 2 — `or "ага"` всегда истинно, программа на любой ввод отвечает «Договорились!».
**Правильно:** `if answer == "да" or answer == "ага":`
**Объяснение:** каждое условие вокруг `or` должно быть полным сравнением. Непустая строка сама по себе — «правда» для Python.

### Урок 3.8 — in: проверка вхождения
**Экраны:** теория → допиши → задача

#### 📖 Теория
Оператор `in` проверяет, есть ли кусочек в строке:

```python
message = "Сегодня будет дождь и ветер"
if "дождь" in message:
    print("Бери зонт")
```

Коротко и читается как английское предложение. А `not in` — наоборот:

```python
if "кот" not in message:
    print("Про котов ни слова")
```

#### ✏️ Допиши код
**Условие:** спам-фильтр! Если в письме есть слово `приз` — помечаем как спам.
**Стартовый код:**
```python
email = input()
if "приз" ___ email:
    print("СПАМ")
else:
    print("Письмо")
```
**Подсказки:**
1. Оператор проверки «содержится ли внутри».
2. Это короткое английское слово `in`.
3. Решение: `if "приз" in email:`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `вы выиграли приз нажмите сюда` | `СПАМ` |
| `привет как дела` | `Письмо` |

#### 💻 Задача
**Условие:** проверка надёжности пароля (наивная). Прочитай пароль. Если в нём меньше 8 символов — `Слабый`. Если 8+ символов, но в нём есть слово `123` — тоже `Слабый`. Иначе — `Нормальный`. Длина строки — `len(password)`.
**Подсказки:**
1. Два условия слабости можно объединить через `or`.
2. `len(password) < 8 or "123" in password`.
3. Решение:
```python
password = input()
if len(password) < 8 or "123" in password:
    print("Слабый")
else:
    print("Нормальный")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `кот` | `Слабый` |
| `суперпароль123` | `Слабый` |
| `надёжный_пёс` | `Нормальный` |

### Урок 3.9 — Марафон повторения
**Экраны:** квиз → баг → задача

#### ❓ Квиз
Что выведет код?
```python
n = 7
if n > 5:
    print("A")
if n > 1:
    print("B")
else:
    print("C")
```
- `A` — ❌ это два НЕЗАВИСИМЫХ if: второй тоже проверится.
- ✅ `A` ⏎ `B`
- `A` ⏎ `B` ⏎ `C` — ❌ else относится ко второму if, а тот истинен — C не выведется.
- `B` — ❌ первый if тоже истинен: 7 > 5.

**Объяснение:** `if`-`if` — проверяются оба; `if`-`elif` — только до первого истинного. Почувствуй разницу!

#### 🐞 Найди баг
**Код:**
```python
mood = input()
if mood == "грустно":
print("Держи печеньку")
```
**Ошибка:** строка 2 — нет отступа.
**Вывод Python:** `IndentationError: expected an indented block after 'if' statement`
**Правильно:** `    print("Держи печеньку")` — 4 пробела.

#### 💻 Задача
**Условие:** игра «камень-ножницы-бумага», ход компьютера всегда `камень`. Прочитай ход игрока и выведи: `бумага` → `Ты победил`, `камень` → `Ничья`, `ножницы` → `Ты проиграл`, всё остальное → `Так ходить нельзя`.
**Подсказки:**
1. Цепочка if/elif/elif/else по значению строки.
2. Сравнивай строки через `==`, последняя ветка else ловит весь мусор.
3. Решение:
```python
move = input()
if move == "бумага":
    print("Ты победил")
elif move == "камень":
    print("Ничья")
elif move == "ножницы":
    print("Ты проиграл")
else:
    print("Так ходить нельзя")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `бумага` | `Ты победил` |
| `камень` | `Ничья` |
| `ножницы` | `Ты проиграл` |
| `пушка` | `Так ходить нельзя` |

### Урок 3.10 — 🏆 БОСС: Текстовый квест
**Экраны:** босс (50 XP)

#### 📖 Вступление
🐍 Пай: «Ты в пещере дракона. Твои решения определят финал. Напиши игру — а потом сыграй в неё сам!»

#### 💻 Задание
**Условие:** квест с двумя развилками.

Программа выводит: `Ты в пещере. Куда пойдёшь? (налево/направо)` и читает ответ.
- `налево` → выводит `Ты нашёл сундук. Открыть? (да/нет)`, читает ответ:
  - `да` → `В сундуке золото! ПОБЕДА`
  - всё остальное → `Осторожность — тоже победа. КОНЕЦ`
- `направо` → выводит `Перед тобой спящий дракон! Бежать? (да/нет)`, читает ответ:
  - `да` → `Ты сбежал. Скучно, но жив. КОНЕЦ`
  - всё остальное → `Дракон проснулся. ПОРАЖЕНИЕ`
- любой другой ответ → `Ты застыл на месте. КОНЕЦ`

**Подсказки:**
1. Внешний if/elif/else по первому ответу, внутри каждой ветки — ещё один if/else по второму. Следи за отступами: внутренние print сдвинуты на 8 пробелов.
2. Внимание: при ответе не «налево/направо» второй input НЕ читается — ветка else внешнего if сразу печатает финал.
3. Решение:
```python
print("Ты в пещере. Куда пойдёшь? (налево/направо)")
way = input()
if way == "налево":
    print("Ты нашёл сундук. Открыть? (да/нет)")
    answer = input()
    if answer == "да":
        print("В сундуке золото! ПОБЕДА")
```
