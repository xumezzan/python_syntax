### Урок 2.1 — Что такое JSON
**Экраны:** теория → квиз

#### 📖 Теория
**JSON** — это текстовый формат, в котором программы пересылают данные друг другу. Человек его тоже легко читает.

```json
{
  "payment_id": 123,
  "amount": 500000,
  "currency": "UZS",
  "status": "pending"
}
```

Правила простые:
- данные — это пары `"ключ": значение`;
- строки — в двойных кавычках `"..."`;
- числа — без кавычек: `500000`;
- значения бывают: строка, число, `true`/`false`, `null`, список `[...]`, объект `{...}`.

Аналогия: JSON — это анкета. Слева вопрос (ключ), справа ответ (значение). И клиент, и сервер заполняют одну и ту же анкету, поэтому понимают друг друга.

🐍 Пай: «99% современных API говорят на JSON. Выучишь его — и поймёшь почти любой ответ сервера».

#### ❓ Квиз
Что из этого — корректный JSON?
- ✅ `{"amount": 500000, "currency": "UZS"}`
- `{amount: 500000}` — ключ обязан быть в двойных кавычках.
- `{'currency': 'UZS'}` — в JSON только двойные кавычки, не одинарные.
- `{"amount": 500000,}` — лишняя запятая в конце недопустима.

### Урок 2.2 — JSON и Python dict
**Экраны:** теория → задача

#### 📖 Теория
JSON и Python-словарь — почти близнецы:

```text
Python dict  ≈  JSON object   {"a": 1}
Python list  ≈  JSON array     [1, 2, 3]
```

В Python есть модуль `json`. Он переводит текст JSON в обычный словарь:

```python
import json

text = '{"status": "paid"}'
data = json.loads(text)   # loads = load string → dict
print(data["status"])     # paid
```

`json.loads` берёт **строку** и возвращает **словарь** (или список). Дальше ты работаешь с ним как с любым Python-объектом.

🐍 Пай: «`loads` — „распаковать строку в данные“. Запомни: l-o-a-d-s, буква s в конце — от string».

#### 💻 Задача
**Условие:** тебе на вход приходит JSON-строка платежа. Считай её, распарси и выведи значение поля `status`.
**Стартовый код:**
```python
import json
text = input()  # например: {"payment_id": 1, "status": "paid"}
# выведи статус
```
**Подсказки:**
1. Преврати строку в словарь через `json.loads(text)`.
2. Достань поле: `data["status"]`.
3. Решение:
```python
import json
text = input()
data = json.loads(text)
print(data["status"])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `{"payment_id": 1, "status": "paid"}` | `paid` |
| `{"status": "failed", "amount": 100}` | `failed` |
| `{"status": "pending"}` | `pending` |

### Урок 2.3 — Вложенные данные
**Экраны:** теория → задача

#### 📖 Теория
Внутри JSON может лежать другой объект или список. Тогда мы спускаемся по уровням, ключ за ключом:

```python
import json

text = '{"order": {"id": 7, "customer": {"name": "Иван"}}}'
data = json.loads(text)
print(data["order"]["customer"]["name"])   # Иван
```

Списки достаём по индексу:

```python
data = json.loads('{"items": ["хлеб", "молоко"]}')
print(data["items"][0])   # хлеб
```

**Ошибка новичка:** пытаться достать вложенное поле одним ключом `data["name"]`, когда оно лежит глубже. Читай структуру сверху вниз — как папки на диске.

🐍 Пай: «Вложенный JSON — это матрёшка. Открываешь по одной кукле за раз».

#### 💻 Задача
**Условие:** приходит JSON заказа с вложенным клиентом: `{"order": {"customer": {"name": "Иван"}}}`. Выведи имя клиента.
**Стартовый код:**
```python
import json
text = input()
# выведи имя клиента
```
**Подсказки:**
1. Спускайся по ключам: сначала `order`, потом `customer`, потом `name`.
2. `data["order"]["customer"]["name"]`.
3. Решение:
```python
import json
data = json.loads(input())
print(data["order"]["customer"]["name"])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `{"order": {"customer": {"name": "Иван"}}}` | `Иван` |
| `{"order": {"customer": {"name": "Anna"}}}` | `Anna` |

### Урок 2.4 — Сериализация и десериализация
**Экраны:** теория → задача → квиз

#### 📖 Теория
Два слова, которые звучат страшно, а значат простое:
- **Десериализация** — текст JSON → Python-объект. Это `json.loads`.
- **Сериализация** — Python-объект → текст JSON. Это `json.dumps` (dump string).

```python
import json

data = {"name": "Ручка", "price": 5}
text = json.dumps(data, ensure_ascii=False)
print(text)   # {"name": "Ручка", "price": 5}
```

Зачем сериализовать? Чтобы **отправить** данные. По сети едет текст, а не Python-объект. Поэтому перед отправкой словарь всегда превращают в строку JSON.

**Ошибка новичка:** забыть `ensure_ascii=False`. По умолчанию `json.dumps` экранирует кириллицу в `Р...` — для русских данных почти всегда добавляют `ensure_ascii=False`, чтобы текст остался читаемым.

🐍 Пай: «loads — входящее письмо распаковать. dumps — своё письмо упаковать перед отправкой».

#### 💻 Задача
**Условие:** собери словарь товара `{"name": "Ручка", "price": 5}` и выведи его как JSON-строку. Чтобы кириллица не превратилась в `\uXXXX`, добавь `ensure_ascii=False`.
**Стартовый код:**
```python
import json
# собери словарь и выведи как JSON (ensure_ascii=False)
```
**Подсказки:**
1. Создай словарь с ключами `name` и `price`.
2. `print(json.dumps(data, ensure_ascii=False))`.
3. Решение:
```python
import json
data = {"name": "Ручка", "price": 5}
print(json.dumps(data, ensure_ascii=False))
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `{"name": "Ручка", "price": 5}` |

#### ❓ Квиз
Перед отправкой данных по сети мы вызываем `json.dumps(data)`. Зачем?
- ✅ `Чтобы превратить Python-словарь в строку JSON для передачи`
- `Чтобы прочитать ответ сервера` — чтение это `loads`, а не `dumps`.
- `Чтобы удалить данные` — `dumps` ничего не удаляет, только превращает в текст.
- `Чтобы ускорить программу` — дело не в скорости, а в формате передачи.

### Урок 2.5 — Отсутствующие поля
**Экраны:** теория → задача → баг

#### 📖 Теория
Реальные ответы не всегда содержат все поля. Если обратиться по ключу, которого нет, — ошибка `KeyError`, и программа падает.

Безопасный способ — метод `.get()` со значением по умолчанию:

```python
data = {"status": "paid"}
print(data.get("currency", "UZS"))   # ключа нет → вернётся UZS
print(data["currency"])              # KeyError! программа упадёт
```

В интеграциях это критично: банк может не прислать поле `error_message`, если ошибки нет. Код должен это пережить, а не упасть.

🐍 Пай: «`.get(ключ, запасное)` — твой ремень безопасности от KeyError».

#### 💻 Задача
**Условие:** приходит JSON платежа, но поля `currency` может не быть. Выведи валюту, а если её нет — `UZS` по умолчанию.
**Стартовый код:**
```python
import json
data = json.loads(input())
# выведи валюту или UZS, если поля нет
```
**Подсказки:**
1. Используй `.get("currency", "UZS")`.
2. Это вернёт значение по умолчанию, если ключа нет.
3. Решение:
```python
import json
data = json.loads(input())
print(data.get("currency", "UZS"))
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `{"amount": 100, "currency": "USD"}` | `USD` |
| `{"amount": 100}` | `UZS` |

#### 🐞 Найди баг
**Код:**
```python
import json
data = json.loads('{"amount": 100}')
print(data["currency"])
```
**Ошибка:** строка 3 — поля `currency` в данных нет, `data["currency"]` бросит `KeyError`.
**Правильно:** `print(data.get("currency", "UZS"))`
**Объяснение после решения:** если поле может отсутствовать, всегда обращайся через `.get()` со значением по умолчанию — иначе программа упадёт на чужих данных.

### Урок 2.6 — Проверка типов
**Экраны:** теория → квиз

#### 📖 Теория
JSON-числа становятся `int`/`float`, строки — `str`, `true`/`false` — `bool`, `null` — `None`.

```python
import json
data = json.loads('{"amount": 500, "paid": true, "note": null}')
print(type(data["amount"]))   # <class 'int'>
print(data["paid"])           # True
print(data["note"])           # None
```

**Ошибка новичка:** считать, что `amount` придёт строкой `"500"`, и сравнивать с текстом. Проверяй: число это число, и складывать `"500" + 100` нельзя.

🐍 Пай: «JSON сам подсказывает тип: есть кавычки — строка, нет — число или булево».

#### ❓ Квиз
В JSON пришло `"paid": false`. Чем это станет в Python после `json.loads`?
- ✅ `Булевым значением False`
- `Строкой "false"` — без кавычек в JSON это не строка.
- `Числом 0` — false не превращается в число автоматически.
- `None` — None получается только из JSON `null`.

### Урок 2.7 — 🏆 БОСС: ответ платёжной системы
**Экраны:** теория → задача

#### 📖 Теория
Платёжный шлюз ответил на твой запрос. Нужно понять итог. Ответ:

```json
{"transaction_id": "TX-9", "result": {"status": "success", "amount": 500000}}
```

Тебе нужен статус из вложенного объекта `result`. Это типичная задача интегратора: пришёл ответ — достань из него главное.

🐍 Пай: «Распарсить чужой ответ и достать нужное поле — половина работы backend-интегратора».

#### 💻 Задача
**Условие:** считай JSON-ответ платёжной системы и выведи `transaction_id` и статус из вложенного `result` — через пробел. Для примера выше: `TX-9 success`.
**Стартовый код:**
```python
import json
data = json.loads(input())
# выведи transaction_id и статус через пробел
```
**Подсказки:**
1. `transaction_id` лежит на верхнем уровне, статус — внутри `result`.
2. `data["transaction_id"]` и `data["result"]["status"]`.
3. Решение:
```python
import json
data = json.loads(input())
print(data["transaction_id"], data["result"]["status"])
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `{"transaction_id": "TX-9", "result": {"status": "success", "amount": 500000}}` | `TX-9 success` |
| `{"transaction_id": "TX-1", "result": {"status": "failed", "amount": 0}}` | `TX-1 failed` |
