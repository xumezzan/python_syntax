### Урок 9.1 — Читаем traceback
**Экраны:** теория → квиз

#### 📖 Теория
Когда программа падает, Python оставляет предсмертную записку — **traceback**:

```
Traceback (most recent call last):
  File "main.py", line 3, in <module>
    print(10 / count)
ZeroDivisionError: division by zero
```

Читай снизу вверх:
- **последняя строка** — ЧТО случилось: тип ошибки и пояснение;
- **строки выше** — ГДЕ: файл, номер строки, сам код.

Ошибка — не «программа сломалась навсегда», а точный отчёт: что и где пошло не так. Программисты не боятся traceback — они его читают. С этого урока ты тоже.

#### ❓ Квиз
Программа упала с таким traceback:
```
  File "main.py", line 7, in <module>
    price = int(text)
ValueError: invalid literal for int() with base 10: 'дорого'
```
Что произошло?
- Программа не нашла файл main.py — ❌ файл найден, он выполнялся.
- ✅ В строке 7 попытались превратить слово «дорого» в число
- Забыли двоеточие в строке 7 — ❌ это была бы SyntaxError.
- Деление на ноль — ❌ это ZeroDivisionError, а тут ValueError.

### Урок 9.2 — Зоопарк ошибок
**Экраны:** теория → квиз → баг

#### 📖 Теория
Познакомься с самыми частыми обитателями traceback. Большинство — твои старые знакомые:

| Ошибка | Когда | Пример |
|--------|-------|--------|
| `SyntaxError` | Python не понял код | забыл `:` или скобку |
| `NameError` | имя не существует | опечатка в переменной |
| `TypeError` | операция не для этого типа | `5 + "5"` |
| `ValueError` | тип верный, значение негодное | `int("абв")` |
| `IndexError` | индекс за пределами | `[1, 2][5]` |
| `KeyError` | нет такого ключа | `d["нет_меня"]` |
| `ZeroDivisionError` | деление на ноль | `x / 0` |

`SyntaxError` — особый: программа даже не начнёт работать. Остальные случаются на лету — и их можно ловить (следующий урок!).

#### ❓ Квиз
Какая ошибка возникнет: `nums = [10, 20, 30]` и затем `print(nums[3])`?
- `KeyError` — ❌ KeyError — про словари, тут список.
- ✅ `IndexError`
- `ValueError` — ❌ дело не в значении, а в выходе за границы.
- Никакая, выведет 30 — ❌ индексы 0,1,2 — тройки нет!

#### 🐞 Найди баг
**Код:**
```python
budgets = {"еда": 5000, "кино": 1500}
print(budgets["такси"])
```
**Ошибка:** строка 2 — ключа `такси` нет → `KeyError: 'такси'`.
**Правильно:** `budgets.get("такси", 0)` — или сначала проверить `in`.
**Объяснение:** модуль 7 уже дал тебе защиту от KeyError — get. Теперь ты знаешь, ОТ ЧЕГО она защищает.

### Урок 9.3 — try/except: подушка безопасности
**Экраны:** теория → допиши → задача

#### 📖 Теория
До сих пор ошибка убивала программу. Блок `try/except` ловит её на лету:

```python
try:
    age = int(input())
    print(f"Через год тебе {age + 1}")
except:
    print("Это не похоже на число")
```

Как работает: Python пробует выполнить блок `try`. Всё гладко — `except` пропускается. Случилась ошибка — выполнение мгновенно прыгает в `except`, и программа живёт дальше.

Ввёл `25` → «Через год тебе 26». Ввёл `двадцать пять` → не падение, а вежливое «Это не похоже на число».

#### ✏️ Допиши код
**Условие:** защити деление от нуля.
**Стартовый код:**
```python
a = int(input())
b = int(input())
___:
    print(a / b)
___:
    print("На ноль делить нельзя!")
```
**Подсказки:**
1. Два ключевых слова: «попробуй» и «а если поймал».
2. `try:` и `except:`.
3. Решение: `try:` перед print(a / b), `except:` перед сообщением.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `10` ⏎ `2` | `5.0` |
| `10` ⏎ `0` | `На ноль делить нельзя!` |

#### 💻 Задача
**Условие:** безопасный конвертер: прочитай строку. Если это число — выведи его квадрат, иначе — `Нужно число`.
**Подсказки:**
1. int(input()) внутри try.
2. Квадрат тоже считай внутри try — после удачного превращения.
3. Решение:
```python
try:
    n = int(input())
    print(n ** 2)
except:
    print("Нужно число")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `7` | `49` |
| `семь` | `Нужно число` |
| `-3` | `9` |

### Урок 9.4 — Ловим конкретную ошибку
**Экраны:** теория → квиз → задача

#### 📖 Теория
Голый `except:` ловит ВСЁ подряд — даже ошибки, о которых ты не подозреваешь. Это как лечить любую болезнь одной таблеткой. Профессионалы указывают тип:

```python
try:
    price = int(input())
    per_item = 100 / price
except ValueError:
    print("Введите число!")
except ZeroDivisionError:
    print("Ноль не подойдёт")
```

Каждый `except` — своя реакция на свою беду. Неожиданные ошибки при этом останутся видимыми — и это хорошо: о них лучше знать, чем молча проглатывать.

#### ❓ Квиз
Что выведет код при вводе `0`?
```python
try:
    x = int(input())
    print(10 // x)
except ValueError:
    print("А")
except ZeroDivisionError:
    print("Б")
```
- `А` — ❌ «0» прекрасно превращается в int — ValueError не было.
- ✅ `Б`
- `0` — ❌ 10 // 0 — это деление на ноль, до print не дошло.
- Программа упадёт — ❌ ZeroDivisionError пойман вторым except.

#### 💻 Задача
**Условие:** доступ к списку победителей: `winners = ["Ия", "Лев", "Ким"]` (задай в коде). Прочитай строку — номер места. Выведи имя победителя, при нечисловом вводе — `Введите число`, при отсутствующем месте — `Такого места нет`. Места нумеруются с 1!
**Подсказки:**
1. Два except: ValueError и IndexError.
2. Место → индекс: `winners[place - 1]`. Учти: отрицательные места тоже должны давать «Такого места нет» — проверь `place < 1` и выбрось за границы сам: `if place < 1: print(...)` внутри try после int, либо добавь проверку.
3. Решение:
```python
winners = ["Ия", "Лев", "Ким"]
try:
    place = int(input())
    if place < 1:
        print("Такого места нет")
    else:
        print(winners[place - 1])
except ValueError:
    print("Введите число")
except IndexError:
    print("Такого места нет")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `1` | `Ия` |
| `5` | `Такого места нет` |
| `два` | `Введите число` |
| `0` | `Такого места нет` |

### Урок 9.5 — else и finally
**Экраны:** теория → квиз

#### 📖 Теория
У try есть ещё два необязательных блока:

```python
try:
    n = int(input())
except ValueError:
    print("Не число")
else:
    print(f"Отлично, {n}!")   # выполнится, только если ошибки НЕ было
finally:
    print("Опрос завершён")    # выполнится ВСЕГДА
```

- `else` — «если всё прошло гладко»: код, которому нечего делать в try (меньше кода под защитой — точнее ловля).
- `finally` — «в любом случае»: уборка, которая должна случиться при любом исходе. Пригодится в модуле 10 с файлами.

Используются реже, чем try/except, но знать их — признак хорошего тона.

#### ❓ Квиз
Что выведет код при вводе `привет`?
```python
try:
    x = int(input())
except ValueError:
    print("ошибка")
else:
    print("успех")
finally:
    print("конец")
```
- `ошибка` — ❌ finally выполняется всегда, его не обойти.
- ✅ `ошибка` ⏎ `конец`
- `успех` ⏎ `конец` — ❌ «привет» в int не превращается — был ValueError.
- `ошибка` ⏎ `успех` ⏎ `конец` — ❌ else выполняется только БЕЗ ошибки.

### Урок 9.6 — Спрашивай, пока не ответят
**Экраны:** теория → задача

#### 📖 Теория
Коронный приём интерактивных программ — повторять вопрос, пока не получишь годный ответ:

```python
while True:
    try:
        age = int(input())
        break              # получилось — выходим!
    except ValueError:
        print("Цифрами, пожалуйста")
print(f"Принято: {age}")
```

Цикл `while True` + `try` + `break` при успехе. Ошибся человек — не страшно, спросим ещё раз. Программы, которые не падают от кривого ввода, — это уважение к пользователю.

#### 💻 Задача
**Условие:** «упрямый кассир»: спрашивай сумму, пока не введут корректное ПОЛОЖИТЕЛЬНОЕ число. На нечисло отвечай `Это не сумма`, на число <= 0 — `Сумма должна быть больше нуля`. Когда всё хорошо — выведи `Чек на X руб.` и закончи.
**Подсказки:**
1. Каркас из теории + дополнительный if внутри try.
2. break ставь только там, где сумма прошла обе проверки.
3. Решение:
```python
while True:
    try:
        total = int(input())
        if total <= 0:
            print("Сумма должна быть больше нуля")
        else:
            break
    except ValueError:
        print("Это не сумма")
print(f"Чек на {total} руб.")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `500` | `Чек на 500 руб.` |
| `дорого` ⏎ `-5` ⏎ `300` | `Это не сумма` ⏎ `Сумма должна быть больше нуля` ⏎ `Чек на 300 руб.` |

### Урок 9.7 — Марафон повторения
**Экраны:** квиз → баг → задача

#### ❓ Квиз
Что выведет код при вводе `abc`?
```python
try:
    n = int(input())
    print("раз")
    print(100 / n)
    print("два")
except ValueError:
    print("три")
```
- `раз` ⏎ `три` — ❌ int(input()) упал ДО первого print.
- ✅ `три`
- `три` ⏎ `два` — ❌ после прыжка в except в try не возвращаются.
- Программа упадёт — ❌ ValueError пойман.

#### 🐞 Найди баг
**Код:**
```python
try:
    n = int(input())
    print(10 / n)
except ValueError:
    print("Не число")
```
**Вопрос:** какой ввод уронит эту программу?
**Ответ:** `0` — ZeroDivisionError никто не ловит.
**Правильно:** добавить `except ZeroDivisionError: print("Не ноль")`.
**Объяснение:** except ловит только указанный тип. Проверяй свой try вопросом: «какие ЕЩЁ ошибки возможны внутри?»

#### 💻 Задача
**Условие:** калькулятор скидки. Прочитай две строки: цену и процент скидки. Выведи цену со скидкой (целое число, `//`). Любая проблема (не числа, процент не в диапазоне 0–100) → одна строка `Ошибка в данных`.
**Подсказки:**
1. Всё в try; диапазон процента проверь if-ом и выведи ошибку в той же ветке.
2. Чтобы из if попасть в «Ошибка в данных» без дублирования строки, можно искусственно вызвать ошибку: `int("x")` — или просто напечатать то же сообщение.
3. Решение:
```python
try:
    price = int(input())
    percent = int(input())
    if percent < 0 or percent > 100:
        print("Ошибка в данных")
    else:
        print(price * (100 - percent) // 100)
except ValueError:
    print("Ошибка в данных")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `1000` ⏎ `25` | `750` |
| `1000` ⏎ `150` | `Ошибка в данных` |
| `дорого` ⏎ `10` | `Ошибка в данных` |

### Урок 9.8 — 🏆 БОСС: Неубиваемая анкета
**Экраны:** босс (50 XP)

#### 📖 Вступление
🐍 Пай: «Финальное испытание на прочность. Анкета, которую НЕВОЗМОЖНО сломать. Я буду вводить всякую дичь — а она выстоит».

#### 💻 Задание
**Условие:** анкета спрашивает три вещи, каждую — до победного:
1. **Имя**: непустая строка (после strip). Пустая → `Имя не может быть пустым`, спросить снова.
2. **Возраст**: целое от 1 до 120. Не число → `Возраст — это число`; вне диапазона → `Не верю!`; спросить снова.
3. **Рост в см**: число с точкой или без (float). Не число → `Рост — это число`; спросить снова.

В конце выведи ровно одну строку: `Имя: X, возраст: Y, рост: Z` (рост — как введён, через float: `170.0`).

**Пример сессии:**
| Ввод | Реакция |
|------|---------|
| `` (пусто) | `Имя не может быть пустым` |
| `Ян` | — |
| `сто` | `Возраст — это число` |
| `999` | `Не верю!` |
| `25` | — |
| `сто70` | `Рост — это число` |
| `170` | `Имя: Ян, возраст: 25, рост: 170.0` |

**Подсказки:**
1. Три цикла while True подряд — по одному на вопрос. Каждый со своим try (где нужен) и break.
2. Имя: `name = input().strip()`, проверка `if name == ""`. Возраст: try + диапазон. Рост: `float(input())` в try.
3. Решение:
```python
while True:
    name = input().strip()
    if name == "":
        print("Имя не может быть пустым")
    else:
        break

while True:
    try:
        age = int(input())
        if age < 1 or age > 120:
            print("Не верю!")
        else:
            break
    except ValueError:
        print("Возраст — это число")

while True:
    try:
        height = float(input())
        break
    except ValueError:
        print("Рост — это число")

print(f"Имя: {name}, возраст: {age}, рост: {height}")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `Ян` ⏎ `25` ⏎ `170` | `Имя: Ян, возраст: 25, рост: 170.0` |
| `` ⏎ `Ия` ⏎ `сто` ⏎ `999` ⏎ `30` ⏎ `привет` ⏎ `165.5` | `Имя не может быть пустым` ⏎ `Возраст — это число` ⏎ `Не верю!` ⏎ `Рост — это число` ⏎ `Имя: Ия, возраст: 30, рост: 165.5` |

**После победы:** +50 XP, бейдж **«Неубиваемый»**. Бонус-режим: платформа запускает «обезьяний тест» — 20 случайных строк мусора в анкету ученика; выстояла — анимация щита.
