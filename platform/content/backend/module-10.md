### Урок 10.1 — REST против SOAP
**Экраны:** теория → квиз

#### 📖 Теория
Ты уже знаешь REST + JSON. Но в мире корпораций живёт ещё один стиль — **SOAP**, и он часто работает с **XML**.

- **REST** — лёгкий, современный, обычно JSON. Большинство новых API.
- **SOAP** — строгий протокол со своим конвертом, обычно XML. Старые и корпоративные системы: банки, страховые, госсервисы, многие модули SAP.

SOAP никуда не делся: если интегрируешься с банком или ERP, шанс встретить XML и SOAP очень высок. Поэтому backend-интегратор обязан уметь их читать.

🐍 Пай: «REST/JSON — это смартфон. SOAP/XML — надёжный кнопочный телефон корпораций. Оба ещё в строю».

#### ❓ Квиз
Почему backend-разработчику в интеграциях важно знать SOAP и XML?
- ✅ `Банки и корпоративные системы (включая SAP) часто общаются по SOAP/XML`
- `SOAP быстрее REST` — дело не в скорости, а в том, где он используется.
- `JSON устарел` — нет, JSON прекрасно живёт, просто не везде.
- `XML нельзя распарсить в Python` — можно, для этого есть готовые модули.

### Урок 10.2 — Что такое XML
**Экраны:** теория → задача

#### 📖 Теория
**XML** — текстовый формат, как JSON, но на тегах (как в HTML). Данные оборачивают в открывающий и закрывающий теги:

```xml
<payment>
    <id>123</id>
    <amount>500000</amount>
    <currency>UZS</currency>
</payment>
```

- `<payment>...</payment>` — элемент с вложенными элементами;
- у тега есть открывающая `<id>` и закрывающая `</id>` часть;
- бывают **атрибуты**: `<payment currency="UZS">`.

В Python XML парсят через `xml.etree.ElementTree`:

```python
import xml.etree.ElementTree as ET
root = ET.fromstring(xml_text)
print(root.find("amount").text)   # 500000
```

🐍 Пай: «XML — это JSON в скобочках-тегах. Те же данные, другая обёртка».

#### 💻 Задача
**Условие:** тебе приходит XML платежа в одну строку. Распарси его и выведи значение `status`.
**Стартовый код:**
```python
import xml.etree.ElementTree as ET
xml = input()  # <payment><id>1</id><status>success</status></payment>
# выведи статус
```
**Подсказки:**
1. Разбери строку: `root = ET.fromstring(xml)`.
2. Найди элемент и возьми текст: `root.find("status").text`.
3. Решение:
```python
import xml.etree.ElementTree as ET
xml = input()
root = ET.fromstring(xml)
print(root.find("status").text)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `<payment><id>1</id><status>success</status></payment>` | `success` |
| `<payment><id>2</id><status>failed</status></payment>` | `failed` |

### Урок 10.3 — Устройство SOAP
**Экраны:** теория → пазл

#### 📖 Теория
**SOAP-сообщение** — это XML с обязательной структурой-«конвертом»:

```xml
<Envelope>
    <Header>...служебное: токены, подписи...</Header>
    <Body>...сами данные запроса/ответа...</Body>
</Envelope>
```

- **Envelope** — внешний конверт, оборачивает всё;
- **Header** — служебная информация (необязателен);
- **Body** — главное: данные операции или ответ.

Ещё есть **WSDL** — это «инструкция» SOAP-сервиса: какие операции он умеет и какие данные принимает. По сути — документация SOAP-API (аналог Swagger у REST).

🐍 Пай: «Envelope — конверт, Header — пометки на конверте, Body — само письмо. WSDL — список, что у сервиса можно спросить».

#### 🧩 Собери код
Расставь части SOAP-сообщения от внешней к внутренней.
**Строки:** `Envelope (конверт)` · `Header (служебное)` · `Body (данные)` · `payment (полезная нагрузка)`
**Правильный порядок:** `Envelope (конверт)` · `Header (служебное)` · `Body (данные)` · `payment (полезная нагрузка)`

### Урок 10.4 — Создание XML-запроса
**Экраны:** теория → квиз → задача

#### 📖 Теория
Чтобы отправить SOAP-запрос, XML часто собирают как строку по шаблону:

```python
xml = f"""<payment>
  <amount>{amount}</amount>
  <currency>{currency}</currency>
</payment>"""
```

Это нормальный приём для простых случаев. Для сложных — есть библиотеки (`zeep` для SOAP), которые строят XML по WSDL за тебя.

**Ошибка новичка:** забыть закрыть тег или перепутать имя открывающего и закрывающего. XML строгий: `<id>1</id>`, а не `<id>1</amount>`.

🐍 Пай: «Собрать XML-строку по шаблону — нормально для простого запроса. Главное — закрывай теги парами».

#### ❓ Квиз
Какой XML записан правильно?
- ✅ `<id>1</id>` — открыли и закрыли одним и тем же тегом.
- `<id>1</amount>` — закрывающий тег не совпадает с открывающим.
- `<id>1` — тег не закрыт вовсе.
- `id>1</id>` — у открывающего тега потеряна скобка.

#### 💻 Задача
**Условие:** собери XML тела платежа. На вход — две строки: сумма и валюта. Выведи `<payment><amount>СУММА</amount><currency>ВАЛЮТА</currency></payment>` без переносов строк.
**Стартовый код:**
```python
amount = input()
currency = input()
# собери и выведи XML одной строкой
```
**Подсказки:**
1. Используй f-строку и подставь обе переменные.
2. Следи, чтобы каждый тег закрывался: `<amount>...</amount>`.
3. Решение:
```python
amount = input()
currency = input()
print(f"<payment><amount>{amount}</amount><currency>{currency}</currency></payment>")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `500000` ⏎ `UZS` | `<payment><amount>500000</amount><currency>UZS</currency></payment>` |

### Урок 10.5 — SOAP Fault
**Экраны:** теория → задача → квиз

#### 📖 Теория
Когда SOAP-сервис возвращает ошибку, он кладёт в `Body` специальный элемент **Fault** (аналог 4xx/5xx в REST):

```xml
<Body>
    <Fault>
        <code>402</code>
        <message>insufficient funds</message>
    </Fault>
</Body>
```

Твой код должен проверить: есть ли в ответе `Fault`? Если да — это ошибка, читаем `message`. Если нет — успех, читаем данные.

🐍 Пай: «Fault в SOAP — это как 4xx в REST: знак, что операция не прошла. Сначала проверь Fault, потом радуйся данным».

#### 💻 Задача
**Условие:** в ответе банка может быть `<fault>`. Если он есть — выведи текст `message` из него; если нет — выведи `ok`.
**Стартовый код:**
```python
import xml.etree.ElementTree as ET
root = ET.fromstring(input())
fault = root.find("fault")
# если fault есть — выведи его message, иначе ok
```
**Подсказки:**
1. `root.find("fault")` вернёт элемент или `None`.
2. Проверь `if fault is not None:` и достань `fault.find("message").text`.
3. Решение:
```python
import xml.etree.ElementTree as ET
root = ET.fromstring(input())
fault = root.find("fault")
if fault is not None:
    print(fault.find("message").text)
else:
    print("ok")
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `<response><fault><code>402</code><message>insufficient funds</message></fault></response>` | `insufficient funds` |
| `<response><status>success</status></response>` | `ok` |

#### ❓ Квиз
В SOAP-ответе пришёл элемент `<Fault>`. Что это значит?
- ✅ `Операция не выполнена — это ошибка, аналог 4xx/5xx в REST`
- `Всё прошло успешно` — наоборот, Fault сигнализирует об ошибке.
- `Это служебный заголовок` — заголовок это Header, а не Fault.
- `Так выглядит любой ответ` — успешный ответ Fault не содержит.

### Урок 10.6 — 🏆 БОСС: SOAP-ответ банка
**Экраны:** теория → задача

#### 📖 Теория
Банк ответил SOAP-сообщением. Тебе нужно пробраться через конверт к данным и достать статус платежа из вложенного тела. Это типичная работа интегратора со старой банковской системой.

🐍 Пай: «Распарсить чужой SOAP-ответ и достать из него статус — навык, за который в интеграциях платят».

#### 💻 Задача
**Условие:** банк прислал ответ `<Envelope><Body><payment><id>...</id><status>...</status></payment></Body></Envelope>`. Достань и выведи `id` и `status` платежа через пробел.
**Стартовый код:**
```python
import xml.etree.ElementTree as ET
root = ET.fromstring(input())
# спустись Envelope → Body → payment и выведи id и status
```
**Подсказки:**
1. Спускайся по уровням: `root.find("Body").find("payment")`.
2. У найденного payment возьми `id` и `status` (их `.text`).
3. Решение:
```python
import xml.etree.ElementTree as ET
root = ET.fromstring(input())
payment = root.find("Body").find("payment")
print(payment.find("id").text, payment.find("status").text)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `<Envelope><Body><payment><id>9</id><status>success</status></payment></Body></Envelope>` | `9 success` |
| `<Envelope><Body><payment><id>3</id><status>failed</status></payment></Body></Envelope>` | `3 failed` |
