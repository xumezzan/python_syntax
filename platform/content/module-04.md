### Урок 4.1 — while: повторяй, пока правда
**Экраны:** теория → допиши → задача

#### 📖 Теория
Цикл `while` повторяет блок кода, пока условие истинно:

```python
power = 1
while power < 100:
    print("Качаюсь! Сила:", power)
    power = power * 2
print("Я готов!")
```
```
Качаюсь! Сила: 1
Качаюсь! Сила: 2
Качаюсь! Сила: 4
...
Я готов!
```

Как это работает: проверь условие → если True, выполни блок → снова проверь условие → … → как только False, перепрыгни на код после цикла.

Выглядит как `if`, но возвращается к проверке снова и снова.

#### ✏️ Допиши код
**Условие:** обратный отсчёт от 5 до 1. Допиши условие цикла.
**Стартовый код:**
```python
n = 5
while n ___ 0:
    print(n)
    n -= 1
print("Старт!")
```
**Подсказки:**
1. Печатаем, пока число не дошло до нуля.
2. Условие: n больше нуля.
3. Решение: `while n > 0:` — когда n станет 0, цикл закончится и выведется «Старт!».
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `5` ⏎ `4` ⏎ `3` ⏎ `2` ⏎ `1` ⏎ `Старт!` |

#### 💻 Задача
**Условие:** вклад 1000 рублей растёт на 10% в год (`money = money * 1.1`). Выведи, сколько ПОЛНЫХ лет нужно, чтобы накопить больше 2000. Выведи одно число — количество лет.
**Подсказки:**
1. Заведи счётчик лет и крути цикл, пока money <= 2000.
2. В цикле две строки: умножай деньги и увеличивай счётчик.
3. Решение:
```python
money = 1000
years = 0
while money <= 2000:
    money = money * 1.1
    years += 1
print(years)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `8` |

### Урок 4.2 — Бесконечный цикл
**Экраны:** теория → квиз → баг

#### 📖 Теория
Если условие никогда не станет ложным — цикл не остановится никогда:

```python
n = 1
while n < 10:
    print(n)
    # забыли n += 1 — программа зависнет!
```

Это **бесконечный цикл** — классическая ошибка. Внутри цикла обязательно должно что-то менять переменную из условия.

🐍 Пай: «Если твоя программа "задумалась" навсегда — жми кнопку ⏹ Стоп. Наша песочница сама остановит цикл после миллиона повторов, но в реальной жизни кнопка Стоп — твой друг».

#### ❓ Квиз
Какой из циклов бесконечный?
- ✅
```python
x = 5
while x > 0:
    print(x)
    x += 1
```
- ❌
```python
x = 5
while x > 0:
    print(x)
    x -= 1
```
(этот остановится: x уменьшается до 0)

**Объяснение:** в первом x растёт: 5, 6, 7… — условие `x > 0` будет истинным вечно. Направление изменения должно вести к выходу из цикла!

#### 🐞 Найди баг
**Код:**
```python
countdown = 3
while countdown > 0:
    print(countdown)
print("Пуск!")
```
**Ошибка:** в теле цикла `countdown` не меняется — бесконечный цикл, печатающий `3`.
**Правильно:** добавить `    countdown -= 1` внутрь цикла (с отступом!).
**Объяснение:** проверь каждый while-цикл вопросом: «что здесь приближает условие к False?»

### Урок 4.3 — for и range
**Экраны:** теория → допиши → задача

#### 📖 Теория
Когда заранее известно, сколько раз повторять, удобнее `for`:

```python
for i in range(3):
    print("Отжимание", i)
```
```
Отжимание 0
Отжимание 1
Отжимание 2
```

`range(3)` — это последовательность `0, 1, 2`: три числа, начиная с нуля. Переменная `i` по очереди принимает каждое значение.

Да, Python считает с нуля — привыкай, все программисты так делают. 😄 Сравни с while: не нужен ни счётчик до цикла, ни `+= 1` внутри — for всё делает сам.

#### ✏️ Допиши код
**Условие:** вывести `Ку-ку!` ровно 4 раза.
**Стартовый код:**
```python
for i in range(___):
    print("Ку-ку!")
```
**Подсказки:**
1. Число в range = сколько раз повторится тело цикла.
2. Нужно 4 повтора.
3. Решение: `range(4)` — i пробежит 0,1,2,3 — четыре раза.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `Ку-ку!` ×4 строки |

#### 💻 Задача
**Условие:** прочитай число n и выведи квадраты чисел от 0 до n−1, каждый на своей строке. Ввод `4` → вывод `0`, `1`, `4`, `9`.
**Подсказки:**
1. `for i in range(n):` и печатай квадрат i.
2. Квадрат — это `i * i` или `i ** 2`.
3. Решение:
```python
n = int(input())
for i in range(n):
    print(i ** 2)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `4` | `0` ⏎ `1` ⏎ `4` ⏎ `9` |
| `1` | `0` |

### Урок 4.4 — range с началом и шагом
**Экраны:** теория → квиз → задача

#### 📖 Теория
У `range` есть три формы:

```python
range(5)         # 0, 1, 2, 3, 4
range(2, 6)      # 2, 3, 4, 5      — от 2 ДО 6 (6 не входит!)
range(0, 10, 2)  # 0, 2, 4, 6, 8   — с шагом 2
range(5, 0, -1)  # 5, 4, 3, 2, 1   — шаг назад!
```

Правило, которое все забывают: **правая граница не включается**. `range(2, 6)` заканчивается на 5. Хочешь до 6 включительно — пиши `range(2, 7)`.

#### ❓ Квиз
Что выведет `for i in range(3, 6): print(i)`?
- `3 4 5 6` — ❌ правая граница (6) не включается.
- ✅ `3` ⏎ `4` ⏎ `5`
- `4` ⏎ `5` ⏎ `6` — ❌ левая граница (3) как раз включается.
- `3` ⏎ `6` — ❌ range перечисляет все числа подряд, шаг по умолчанию 1.

#### 💻 Задача
**Условие:** таблица умножения на 7! Выведи 9 строк вида `7 x 1 = 7` … `7 x 9 = 63`.
**Подсказки:**
1. Цикл от 1 до 9 включительно: `range(1, 10)`.
2. Внутри: `print("7 x", i, "=", 7 * i)`.
3. Решение:
```python
for i in range(1, 10):
    print("7 x", i, "=", 7 * i)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `7 x 1 = 7` ⏎ … ⏎ `7 x 9 = 63` (9 строк) |

### Урок 4.5 — Цикл по строке
**Экраны:** теория → допиши → задача

#### 📖 Теория
`for` умеет ходить не только по числам, но и по буквам строки:

```python
for letter in "кот":
    print(letter)
```
```
к
о
т
```

Читается почти по-английски: «для каждой буквы в слове». Имя переменной (`letter`) — любое, Python сам положит в неё очередной символ.

#### ✏️ Допиши код
**Условие:** робот-диктор произносит слово по буквам с восклицанием. Допиши заголовок цикла.
**Стартовый код:**
```python
word = input()
for ch ___ word:
    print(ch + "!")
```
**Подсказки:**
1. То же слово, что в range-циклах, — «в».
2. Оператор `in`.
3. Решение: `for ch in word:`.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `да` | `д!` ⏎ `а!` |
| `ура` | `у!` ⏎ `р!` ⏎ `а!` |

#### 💻 Задача
**Условие:** посчитай, сколько букв `о` в введённой строке. Ввод `молоко` → вывод `3`.
**Подсказки:**
1. Счётчик до цикла, в цикле if по букве.
2. `if ch == "о": count += 1`.
3. Решение:
```python
text = input()
count = 0
for ch in text:
    if ch == "о":
        count += 1
print(count)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `молоко` | `3` |
| `привет` | `0` |
| `ооо` | `3` |

### Урок 4.6 — break: аварийный выход
**Экраны:** теория → квиз → задача

#### 📖 Теория
`break` мгновенно останавливает цикл:

```python
for floor in range(1, 100):
    print("Этаж", floor)
    if floor == 3:
        print("Моя остановка!")
        break
```
```
Этаж 1
Этаж 2
Этаж 3
Моя остановка!
```

Цикл собирался до 99 этажа, но `break` высадил нас на третьем.

Частый приём — «вечный цикл с выходом изнутри»: `while True:` + `break` по условию. Звучит страшно, выглядит элегантно:

```python
while True:
    answer = input()
    if answer == "стоп":
        break
    print("Ты сказал:", answer)
```

#### ❓ Квиз
Что выведет код?
```python
for i in range(10):
    if i == 2:
        break
    print(i)
```
- `0` ⏎ `1` ⏎ `2` — ❌ break срабатывает ДО print на той итерации, где i == 2.
- ✅ `0` ⏎ `1`
- `0` ⏎ `1` ⏎ … ⏎ `9` — ❌ break прерывает цикл насовсем.
- `2` — ❌ наоборот: двойка как раз не печатается.

#### 💻 Задача
**Условие:** попугай-повторюшка: читай строки и выводи каждую обратно, пока не встретишь слово `тишина` — его не выводи, просто остановись.
**Подсказки:**
1. `while True:` + input + проверка + break.
2. Проверку делай ДО печати.
3. Решение:
```python
while True:
    word = input()
    if word == "тишина":
        break
    print(word)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `привет` ⏎ `кеша` ⏎ `тишина` | `привет` ⏎ `кеша` |
| `тишина` | (пусто) |

### Урок 4.7 — continue: пропусти ход
**Экраны:** теория → квиз → допиши

#### 📖 Теория
`continue` не останавливает цикл, а пропускает остаток текущего шага и переходит к следующему:

```python
for day in range(1, 8):
    if day == 6 or day == 7:
        continue          # выходные пропускаем
    print("День", day, "- работаем")
```

`break` — «выйти из автобуса», `continue` — «проехать остановку».

#### ❓ Квиз
Что выведет код?
```python
for i in range(5):
    if i % 2 == 0:
        continue
    print(i)
```
- `0` ⏎ `2` ⏎ `4` — ❌ наоборот: чётные пропускаются через continue.
- ✅ `1` ⏎ `3`
- `1` ⏎ `3` ⏎ `5` — ❌ range(5) заканчивается на 4, пятёрки нет.
- Ничего — ❌ нечётные числа доходят до print.

#### ✏️ Допиши код
**Условие:** выведи все числа от 1 до 10, кроме делящихся на 3.
**Стартовый код:**
```python
for i in range(1, 11):
    if i % 3 == 0:
        ___
    print(i)
```
**Подсказки:**
1. Числа, делящиеся на 3, надо перешагнуть, не останавливая цикл.
2. Не break — он убьёт весь цикл.
3. Решение: `continue` → выведется 1 2 4 5 7 8 10.
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `1` ⏎ `2` ⏎ `4` ⏎ `5` ⏎ `7` ⏎ `8` ⏎ `10` |

### Урок 4.8 — Накопитель
**Экраны:** теория → задача → задача

#### 📖 Теория
Самый главный паттерн циклов — **накопитель**: переменная, которая собирает результат шаг за шагом.

```python
total = 0                  # 1. пустой накопитель ДО цикла
for price in range(1, 5):
    total += price         # 2. пополняем НА КАЖДОМ шаге
print(total)               # 3. результат ПОСЛЕ цикла → 10
```

По этой схеме считают суммы, произведения, количества, склеивают строки. Три места, три роли — запомни эту форму, она будет в каждой второй программе твоей жизни.

#### 💻 Задача 1
**Условие:** прочитай n, затем n чисел (по одному в строке) — это твои траты за день. Выведи сумму.
**Подсказки:**
1. `total = 0`, цикл `for i in range(n)`, внутри читай число и прибавляй.
2. Внутри цикла: `total += int(input())`.
3. Решение:
```python
n = int(input())
total = 0
for i in range(n):
    total += int(input())
print(total)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `3` ⏎ `100` ⏎ `250` ⏎ `50` | `400` |
| `1` ⏎ `7` | `7` |

#### 💻 Задача 2
**Условие:** прочитай n, затем n чисел — результаты прыжков в длину (в см). Выведи лучший (максимальный) результат.
**Подсказки:**
1. Накопитель для максимума: заведи `best = 0` (прыжки не отрицательные) и обновляй, если очередное число больше.
2. `if jump > best: best = jump`.
3. Решение:
```python
n = int(input())
best = 0
for i in range(n):
    jump = int(input())
    if jump > best:
        best = jump
print(best)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `3` ⏎ `150` ⏎ `210` ⏎ `180` | `210` |
| `2` ⏎ `90` ⏎ `90` | `90` |

### Урок 4.9 — Цикл в цикле
**Экраны:** теория → квиз → задача

#### 📖 Теория
Цикл может жить внутри цикла. Внешний делает шаг — внутренний прокручивается целиком:

```python
for row in range(3):
    for col in range(3):
        print("*", end="")
    print()
```
```
***
***
***
```

Две новинки: `end=""` говорит print'у «не переходи на новую строку», а пустой `print()` — переходит. Так и рисуется квадрат: внутренний цикл печатает строку из звёздочек, внешний — переводит строку.

#### ❓ Квиз
Сколько раз напечатается `Бип`?
```python
for i in range(3):
    for j in range(4):
        print("Бип")
```
- 7 — ❌ циклы не складываются, а умножаются.
- ✅ 12
- 3 — ❌ на каждый из 3 шагов внешнего внутренний делает 4 своих.
- 4 — ❌ внешний цикл повторит внутренний трижды.

#### 💻 Задача
**Условие:** прочитай n и нарисуй лесенку из n ступенек:
```
#
##
###
```
(для n=3). В строке i — i символов `#`.
**Подсказки:**
1. Внешний цикл по номерам строк от 1 до n, внутри печатай строку.
2. Можно без вложенного цикла: `print("#" * i)` — умножение строк из модуля 2!
3. Решение:
```python
n = int(input())
for i in range(1, n + 1):
    print("#" * i)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| `3` | `#` ⏎ `##` ⏎ `###` |
| `1` | `#` |

### Урок 4.10 — Марафон повторения
**Экраны:** квиз → баг → задача

#### ❓ Квиз
Что выведет код?
```python
s = 0
for i in range(1, 4):
    s += i
print(s)
```
- `1` ⏎ `2` ⏎ `3` — ❌ print стоит после цикла, без отступа — он один.
- ✅ `6`
- `4` — ❌ суммируются 1+2+3, правая граница не входит.
- `10` — ❌ range(1, 4) — это 1, 2, 3. Четвёрки нет.

#### 🐞 Найди баг
**Код:**
```python
total = 0
for i in range(5):
    total = i
print(total)
```
**Задумка:** сумма чисел 0..4 (ожидали 10).
**Ошибка:** строка 3 — `=` вместо `+=`: каждый шаг затирает накопитель, выйдет 4.
**Правильно:** `total += i`.
**Объяснение:** накопитель пополняют (`+=`), а не перезаписывают (`=`).

#### 💻 Задача
**Условие:** FizzBuzz — легендарное собеседование в твоём первом модуле циклов! Для чисел от 1 до 15: если делится на 3 и на 5 — выведи `FizzBuzz`, только на 3 — `Fizz`, только на 5 — `Buzz`, иначе — само число.
**Подсказки:**
1. Цикл + цепочка if/elif/elif/else. Проверку «на 3 и на 5» ставь ПЕРВОЙ.
2. `if i % 15 == 0` — или `i % 3 == 0 and i % 5 == 0`.
3. Решение:
```python
for i in range(1, 16):
    if i % 15 == 0:
        print("FizzBuzz")
    elif i % 3 == 0:
        print("Fizz")
    elif i % 5 == 0:
        print("Buzz")
    else:
        print(i)
```
**Автотесты:**
| Ввод | Ожидаемый вывод |
|------|------------------|
| — | `1` ⏎ `2` ⏎ `Fizz` ⏎ `4` ⏎ `Buzz` ⏎ … ⏎ `FizzBuzz` (15 строк) |

### Урок 4.11 — 🏆 БОСС: Угадай число
**Экраны:** теория-вставка → босс (50 XP)

#### 📖 Мини-теория: случайные числа
Программа может «загадывать» с помощью модуля random:

```python
import random
secret = random.randint(1, 100)   # случайное целое от 1 до 100
```

`import random` — подключение модуля, пиши его в первой строке. Подробнее про модули — позже, сейчас просто пользуемся.

#### 💻 Задание
**Условие:** игра «Угадай число». Программа загадывает число от 1 до 100 и в цикле:
1. читает попытку игрока;
2. если попытка меньше загаданного — выводит `Больше!`;
3. если больше — `Меньше!`;
4. если угадал — `Победа! Попыток: N` и завершается.

Для автотестов: первой строкой ввода приходит «загаданное» число (вместо random) — в коде для проверки используй `secret = int(input())`. Кнопка «🎮 Играть» в редакторе подменяет эту строку на `random.randint`.

**Подсказки:**
1. `while True:` + if/elif/else внутри + break при победе + счётчик попыток.
2. Считай попытки: `tries += 1` сразу после input.
3. Решение:
```python
secret = int(input())
tries = 0
while True:
    guess = int(input())
    tries += 1
    if guess < secret:
        print("Больше!")
    elif guess > secret:
        print("Меньше!")
    else:
```
